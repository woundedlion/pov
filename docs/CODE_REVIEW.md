# Holosphere & Daydream — Comprehensive Code Quality Review

**Date:** 2026-06-10 (seventh audit; succeeds the sixth audit recorded earlier at tree `391e670`, whose 146 findings are all closed)
**Tree:** `9e5a664`
**Scope:** The full Holosphere repository (engine, effects, hardware drivers, WASM bridge, tests, build/CI, documentation) and the sibling daydream repository (simulator app, segment workers, recorder, geometry tools). Out of scope by instruction: `core/effects_legacy.h`, `targets/Holosphere/Holosphere.ino`, `core/rotate.h`. `core/FastNoiseLite.h` is vendored third-party code and was assessed only for integration.
**Method:** Twelve independent component reviewers each read every in-scope file in their component end-to-end, with the README as shared architectural context, and graded each quality dimension. The prior `CODE_REVIEW.md` was deliberately not consulted during examination; it was reconciled afterward (§6), which removed one finding that re-raised a maintainer-rejected item and reframed another. Finding numbers continue the project's running sequence: this audit's findings are **147–268**.

## 1. Executive Summary

This is the first audit of this codebase to find **zero critical and zero high-severity defects**. The prior audit's entire fix list — including a hardware P0 (the eDMA TCD misconfiguration), the frame-sync protocol gap, and the daydream startup/runtime guard holes — has demonstrably landed: the twelve reviewers on this pass, reading the tree cold, independently confirmed the fixes as *strengths* (the count-coded sync symbol alphabet and its 4-board simulator, the fail-dark doctrine, the transactional WASM boundary, the detach-aware pixel-view healing, on-demand rendering). What remains is a residual profile of 17 medium and 105 low findings over ~62k lines — overwhelmingly latent edge cases, sibling-convention inconsistencies, hot-loop palette discipline lapses, and documentation drift, with not one reproducible user-facing failure on a shipping default path in the C++ engine.

The grade-limiting residuals cluster in three places:

1. **Hot-loop palette discipline.** The codebase has a clear house pattern — bake `GenerativePalette` into a 256-entry LUT before per-fragment use — that four effects (Raymarch, Moire, RingShower, MobiusGrid) ignore, running a ~2000-cycle dual sRGB→OKLCH conversion per rasterized fragment. Moire's Density slider can plausibly blow the 600 MHz device frame budget (finding 156).
2. **The JavaScript frontend's least-tested half.** The deep-link hydration layer — ironically, machinery added to fix prior findings — reintroduced a way for an invalid `?resolution=` URL to brick startup (finding 161), and the sim's display convention is mirrored relative to the engine's documented coordinate system (finding 162). JS test coverage remains inverted relative to risk: `state.js` and the worker pool are well tested; `gui.js`, `driver.js`, and the five tools — where every found JS bug lives — have none.
3. **Device-shell-only hardware paths.** Everything host-testable in `hardware/` is now implemented, simulated, and verified — the protocol core survived adversarial tracing of fold/snap ordering, epoch inference, and counter wrap. The one consequential defect found lives precisely in the untestable shell: `render_black()` latches `dark_latched_` even when the DMA controller drops the black frame, so the fail-dark guarantee can hold a stale bright column for an entire dark window (finding 157).

The deepest structural compliment this audit can pay: the fail-fast doctrine, the arena contracts, the determinism guarantees, and the comment culture are now so uniformly applied that most findings are *internal inconsistencies with the project's own standards* — a missing `HS_CHECK` where every sibling has one, an unwrapped trig phase where two other effects carefully wrap, one mesh routine bypassing the accessor discipline the other thirteen follow. The project's standards have become the sharpest tool for auditing it.

**Verdict: A− overall — a high A−.** The engine core, hardware protocol layer, and test suite each grade A on their own. The distance to a flat A consists of the JS frontend's verification gap, the per-fragment OKLCH lapses, the residual mediums below, and the device-shell code that no host audit can close.

## 4. Component Capsules

**platform-foundations (platform, constants, canvas, effects/registry, led, util, waves, easing, concepts, presets).** Excellent shape. One platform seam with three targets and unusually well-articulated contracts: the HS_CHECK doctrine, the mt19937(1337) determinism contract, the NaN-propagating clamp with operand-order rationale, the double-buffer index protocol with its correctness argument written at the exact point a reviewer would question it, and a Canvas-constructor watchdog so a stalled ISR traps instead of hanging headless. The consequential interface problem is ClipRegion's x-clip API, which forces three hot loops to hand-roll a subtle wrap predicate — two of them already inconsistently (finding 147). Host-mock fidelity claims slightly overreach (1-LSB scale8 divergence, CRGB constructor semantics).

**math-color (3dmath, geometry, color, color_luts, palettes, styles).** The strongest kind of embedded numerics: every fast approximation carries a measured error budget, exact trig is deliberately kept where drift would accumulate, singularity handling is a coherent system (shared STEREO_INF sentinel, `normalized_or` vs trapping `normalized`), and the 16-bit linear pipeline delivers on the README's promise with checked-in generated LUTs and NaN-safe casts before every float→int conversion. The one real bug: `EdgeFadeShade`'s low edge blends in 8-bit sRGB while its high edge blends in 16-bit linear — enabled by the implicit `Pixel16↔CRGB` bridge, and directly contradicting the project's central philosophy (finding 148).

**render-pipeline (filter, sdf, scan, plot, transformers).** The heart of the engine and its strongest part. The variadic Pipeline's compile-time domain transitions, the three-method SDF shape contract, the seam-aware scanline coalescer, the fixed-point delta-encoded feedback warp field, and the per-face distance LUT with a provably sign-pure bound all survived adversarial reading. Defects are latent or second-order: an unclamped Replicate step, mid-buffer expired trail redraws, a device-only AntiAlias/H_OFFSET divergence the host suite structurally cannot see.

**animation-memory (animation, memory, generators, static_circular_buffer, reaction_graph).** The structural backbone, with unusual rigor: overflow-proof subtractive bounds math, layered use-after-free defense (debug generation stamps + release `owns_live` checks + rebind counters), compile-time borrow contracts via deleted rvalue overloads, and a type-erased inline-storage Timeline whose orientation-collapse dedup preserves composed motion blur. Main costs: `animation.h` is a 2,226-line monolith, and the CubemapLUT descent converges only via an accident-of-correctness the comment contradicts (finding 149).

**mesh-system (mesh, conway, hankin, solids, spatial).** Mature and carefully engineered: thirteen Conway operators on shared walk scaffolds with exact upfront capacity sizing, a textbook bake-vs-evaluate Hankin split, a correct embedded-appropriate KDTree, and every narrowing/sentinel boundary converted to a trap with its rationale. The one hole in the fail-fast surface: `compile_hankin` bypasses the unified accessors and would silently corrupt on a borrowed-mode MeshState (finding 150). The load-bearing arena polarity contracts live entirely in (excellent) comments rather than types.

**effects (A–H).** Compact, self-contained compositions with exceptional arena discipline — per-effect `configure_arenas` splits justified with byte math that checks out — and mature live-parameter plumbing. `ReactionDiffusionBase` is a textbook CRTP extraction. No memory-safety, UB, or race defects; findings are second-order (HopfFibration's unwrapped trig phases against the project's own multi-day standard; two README blurbs describing older incarnations of the effects).

**effects (I–Z).** Sophisticated engineering judgment throughout: slot lifetimes as pure functions of age with the dangling-slot rationale documented, Liquid2D's exemplary phase-wrapping hygiene, Voronoi's KD-tree migration with a correct exactness proof. The systemic weakness is per-fragment `GenerativePalette::get` in four effects against the established baking pattern (findings 155, 156, 221, 222), plus one genuinely wrong de-phasing design in MeshFeedback (154).

**hardware-drivers.** The strongest-engineered firmware component this audit series has reviewed. The pure-core/device-shell split means every load-bearing protocol decision is host-tested, including a 1,500-line 4-board simulator with crystal ppm offsets, masked-IRQ models, EMI injection, and reboots. The flywheel timebase makes counter wrap structurally unobservable; the failure asymmetry (fail-to-missed-never-wrong) is carried coherently through every layer; documentation matches the code constant-for-constant. The one defect with real consequence lives in the untestable shell: the fail-dark latch on a dropped black frame (finding 157).

**wasm-bridge & build.** Disciplined boundary engineering: every JS entry point transactional, the zero-copy memory-view contract documented at both ends and enforced by a host test, single-source-of-truth X-macros, provenance-stamped installs, SHA-pinned CI with Windows toolchain coverage. The MeshOps tooling surface is held to a visibly lower standard (unvalidated numerics, the `clearToolingMemory` lifetime trap) — defensible for editor tooling, inconsistent with the file's own doctrine.

**test-suite.** Built and run at HEAD: exit 0, all 26 death traps fire with the exact expected signal. Oracle-based assertions dominate; regression tests pin the exact bug and include discriminating checks that fail on the old code; the effect harness is roster-generated and cannot drift. Gaps: no sanitizer configuration anywhere (the one layer that would cover the suite's own acknowledged arena-lifetime blind spots), a 25 s monolithic run its own comments claim is "a few seconds," three `-Wunused-result` warnings.

**daydream-app.** Unusually well-engineered frontend with a written failure doctrine the code genuinely follows — except in the deep-link hydration layer, the cleverest and least-tested part, where the doctrine's flagship case (invalid user-supplied resolution) actually fails by a route the prior fix itself introduced (finding 161). One genuine convention divergence: the displayed sphere is mirrored relative to the engine's documented coordinate mapping (finding 162). Zero coverage for `gui.js`, `driver.js`, `sidebar.js` — where every found bug lives.

**daydream-segments & tools.** The worker pool is production-quality concurrent JS: generation fencing, a fault latch that deliberately breaks the silent-deadlock mode, promise-chain message serialization, typed contracts, and tests targeted at exactly the riskiest glue. The five geometry tools are a clear tier below: functional and useful, with real but low-stakes defects (a negative-width mobile resize, a wrong tooltip formula, touch-input gaps) and zero automated coverage.

### Medium

147. ✅ **ClipRegion's x-clip API invites the rs==re wrap-to-full bug; two hot-loop consumers already reproduce it** — `core/constants.h:36`
   `render_x_start()/render_x_end()` wrap mod W, so an arc-plus-margins that sums to exactly W yields `rs == re` (meaning full width). `contains_x` special-cases this, but the class exposes no precomputed predicate, so consumers hand-roll it: `filter.h:848` guards correctly, while `scan.h:264` (rasterize) and `scan.h:757` (volume) treat rs==re as empty — a latent full-segment blackout. Expose a precomputed x-clip functor so the special case lives in one place. **Fixed:** added `ClipRegion::x_clip()`, returning an `XClip` functor whose `clipped(x)` predicate folds both the full-width and rs==re wrap-to-full cases into a single `active` flag. The rasterize and volume scan loops and the segmented-feedback prepass now all build the functor once per draw and call `clipped(wx)` instead of hand-rolling the wrap test, so the two scan.h consumers no longer black out a full segment (and the prepass's over-clip on a full-width, y-clipped region is gone too). Inlined, the predicate compiles to the same branch sequence as before — no per-fragment cost.

148. ✅ **EdgeFadeShade's low-edge band blends in 8-bit sRGB, not 16-bit linear** — `core/color.h:1120`
   The `t < edge` branch calls `lerp16` on a CRGB temporary, selecting the 8-bit gamma-space overload (quantize → sRGB lerp → re-expand), while the high-edge branch stays in 16-bit linear via `Pixel16::lerp16` — banding and an asymmetric fade in exactly the soft vignettes the README cites as the 16-bit motivation. Compiled silently because `Pixel16↔CRGB` convert implicitly both ways (see finding 175). Use `Pixel(0,0,0).lerp16(...)`. No production effect currently uses EdgeFadeShade, which caps impact. **Fixed:** the local `black` is now a `Pixel` (16-bit linear) instead of a `CRGB`, so the low edge resolves to `Pixel16::lerp16` and both edges fade symmetrically in 16-bit linear.

149. ✅ **CubemapLUT::find_nearest_node's 64-iteration cap is sufficient only via an undocumented mid-loop mutation that contradicts its own comment** — `core/reaction_graph.h:116`
   The latitude-only seed can leave ~78 hops of geodesic to traverse, exceeding the cap under the documented one-hop-per-iteration algorithm; the code converges only because `cur = ni` is assigned *inside* the k-loop, letting each iteration chain multiple hops. A well-intentioned refactor to a textbook best-of-neighbors step would silently corrupt equatorial LUT entries, and the round-trip test cannot catch it (it seeds at the answer). Document the multi-hop chaining as load-bearing, or raise the cap and select best-of-neighbors. **Fixed:** kept the (correct) descent unchanged and documented it — a `LOAD-BEARING` block on `find_nearest_node` now spells out that the mid-scan `cur`/`best_d` reassignment chains several hops per `iter`, that the 64 cap depends on it because the latitude-only seed can be dozens of hops from an equatorial answer, and that converting to best-of-neighbors requires raising the cap; an inline note flags the deliberate reassignment.

150. ✅ **compile_hankin bypasses the unified mesh accessors; borrowed-mode MeshState input silently corrupts in release builds** — `core/hankin.h:86`
   It reads `mesh.face_counts.size()`/`mesh.faces.size()` directly instead of the `get_*_size()` accessors every conway.h operator uses for exactly this case. A borrowed-mode MeshState computes F=I=0, binds zero-capacity buffers, then `get_midpoint_idx` reads uninitialized arena memory — silently wrong geometry in NDEBUG. No current caller hits it, but the template advertises the same MeshT generality the conway operators honor. The one silent hole in an otherwise watertight fail-fast surface. **Fixed:** F and I now come from `mesh.get_face_counts_size()`/`mesh.get_faces_size()`, matching the conway operators and the `HalfEdgeMesh` ctor this function already feeds; vertices stays a direct read since it is always owned. Both `PolyMesh` and `MeshState` expose the accessors, so the `MeshT` template stays general.

151. ✅ **Comets README blurb describes an implementation that no longer exists** — `README.md:1465`
   "Particles … launched in bursts and influenced by rotational gravity": the code has no particle system, bursts, or gravity — it is a single Lissajous-following head with a 115-frame orientation trail and periodic palette wipes. **Fixed:** the blurb now describes the single head tracing spherical Lissajous curves through a dozen configurations, its 115-frame orientation trail, and the periodic triadic palette wipes.

152. ✅ **ChaoticStrings README blurb claims frequency-ratio sweeping the code does not do** — `README.md:1499`
   The implementation uses one fixed 12:5 Lissajous configuration set at init; nothing sweeps rational approximations. The visual variation comes from noise-transformer jitter and palette cycling. **Fixed:** the blurb now states the fixed 12:5 figure, the noise-transformer warping of the trail, and the cycling gradient palette.

153. ✅ **HopfFibration accumulates three trig-feeding phases unbounded, against the project's own multi-day precision standard** — `effects/HopfFibration.h:50`
   `flow_offset`, `tumble_angle_x`, `tumble_angle_y` advance with `Driver wrap=false` and feed `fast_sinf/cosf`; float ULP growth produces visible stepping over multi-day installation runs. Flyby and DreamBalls wrap the equivalent phases with comments citing precisely this hazard. Accumulate in wrapped turns and scale at use. **Fixed:** the three Drivers now run `wrap=true`, accumulating each phase as a fraction of its natural period in `[0,1)` turns (rate = old radians-per-unit ÷ period), and the phase is scaled back to radians at the use site. `flow_offset`/`tumble_angle_y` wrap over 2π; `tumble_angle_x` wraps over **4π** because it also feeds the half-angle `fold_base` term, so both its full-angle (`cx`/`sx`) and half-angle uses stay continuous across the wrap. The transform is algebraically identical to the unbounded version (the wrapped offsets differ only by whole periods), so the visual is unchanged while the `fast_sinf/cosf` arguments stay bounded.

154. ✅ **MeshFeedback's coprime de-phasing design is arithmetically wrong: the shape cycle equals the preset cycle exactly** — `effects/MeshFeedback.h:15`
   `NO_MORPH_FRAMES=81` is commented "coprime with the 241-frame preset cycle so the two stay out of phase," but the true shape period is 81+160 = 241 — equal to PRESET_FRAMES — so every preset hard-cut lands at the same phase (morph completion) every cycle. The coprimality of 81 and 241 is irrelevant because 81 is not the period. **Fixed:** the period is now an explicit `SHAPE_FRAMES = 240` (coprime with the prime `PRESET_FRAMES = 241`, and `= 241 - 1` so the beat is maximal), with `NO_MORPH_FRAMES = SHAPE_FRAMES - MORPH_FRAMES` deriving the 80-frame hold. The shape and preset cycles now genuinely de-phase (LCM 240·241), and the comment explains that the *period*, not the hold, is the quantity that must be coprime. (The prior comment also wrongly called 81 prime; 81 = 3⁴.)

155. ✅ **Raymarch evaluates an immutable fixed-seed GenerativePalette through the full OKLCH pipeline per shaded fragment** — `effects/Raymarch.h:133`
   ~2000 cycles per call (two sRGB→OKLCH conversions) for every fragment of 20 ray-marched tori, on the roster's most per-pixel-expensive effect — and the palette is constructed with `manual_seed=219` and never mutated, making it trivially bakeable (SphericalHarmonics bakes for exactly this pattern). **Fixed:** added a `BakedPalette` member baked once from the immutable `palette` in `init()`; the per-fragment shader now does a 256-entry LUT lookup (16-bit linear interpolation) instead of the dual sRGB→OKLCH conversion, matching the established baking pattern.

156. ✅ **Moire runs per-fragment OKLCH interpolation; the Density slider can blow the device frame budget** — `effects/Moire.h:96`
   ~26,000 OKLCH interpolations per frame at the WASM default; the registered Density max of 100 reaches ~20,000 calls (tens of milliseconds at 600 MHz) at device resolution. The palettes are wipe-animated, but re-baking a 256-entry LUT once per frame is ~40–100× cheaper than per-fragment evaluation. **Fixed:** both source palettes now bake into `BakedPalette` LUTs, rebaked once per frame in `draw_frame()` (2×256 OKLCH evals, regardless of Density), and `draw_layer`'s fragment shader does a LUT lookup. Per-frame palette cost is now constant instead of scaling with Density × ring fragments.

157. ✅ **Fail-dark guarantee can latch a stale bright column: render_black() marks dark_latched_ even when the black frame is dropped on DMA overrun** — `hardware/pov_segmented.h:479`
   `submitFrame()` silently drops and returns void when the prior column's DMA is in flight; the dark branch sets `dark_latched_ = true` unconditionally and never resubmits, so the strip can hold the last bright column for an entire ACQUIRE or construction window — inverting the design's own missed-never-wrong asymmetry (a dropped image column self-heals in 434 µs; a dropped black frame latches indefinitely). Have `submitFrame()` report acceptance and latch only on success, or resubmit black on every wake while dark. **Fixed:** `DMALEDController::submitFrame()` now returns `true` on accept / `false` on overrun-drop, `render_black()` propagates that result, and the dark branch latches with `if (!dark_latched_ && render_black())`. A dropped black frame now leaves `dark_latched_` false and retries on the next column tick, so the strip goes dark within ~434 µs instead of holding the stale bright column for the whole dark window. Steady-state column submits still ignore the result (a dropped image column self-heals next tick).

158. ✅ **wasm_smoke's high-water-mark assertions are unsatisfiable; a fully saturated stack passes the gate** — `scripts/wasm_smoke.mjs:94`
   The loop fails on `hwm > capacity`, but arenas trap before hwm can exceed capacity and the stack canary scan is capped at capacity by construction — so the stack, the one region with no trap and the test's headline claim (echoed in ci.yml), can at worst report `hwm == capacity` and pass. Fail on `hwm >= capacity` or enforce a margin. **Fixed:** the stack now has a dedicated margin gate (`STACK_MAX_FILL = 0.75`) that fails when its high-water mark reaches 75% of capacity, well before the cliff (current peak ~616/8192 bytes); the arena `> capacity` check is kept as the correct never-fires guard for the self-trapping arenas, where a full arena is legal.

159. ✅ **Declared CMake floor (3.21) is below what the infrastructure requires** — `CMakePresets.json:2`
   Presets schema v6 needs CMake 3.25 (3.21–3.24 reject the file outright), and `CMAKE_LINKER_TYPE` in the native toolchain needs 3.29 — on 3.25–3.28 it is silently ignored and the Windows tests preset falls back to MSVC link.exe discovery, the exact failure the toolchain file exists to prevent. Lower the schema version with a guarded fallback, or raise the declared minimums. **Fixed:** raised the declared floor to 3.29 (the real requirement) in all three sites — `CMakePresets.json` `cmakeMinimumRequired`, the root `CMakeLists.txt`, and `tests/CMakeLists.txt` (which gates the standalone `cmake -S tests` path) — so an old CMake fails loudly instead of mis-linking. Verified both the `tests` and `wasm-release` presets still configure on CMake 4.2.3.

160. ✅ **No sanitizer or warnings configuration for the host test build** — `tests/CMakeLists.txt:34`
   For an arena-allocated, header-only engine whose device build compiles guards out, the native suite is the one place ASan/UBSan could run — and they would directly cover the arena-lifetime blind spots the suite itself acknowledges (finding 244). The absent `-Wall`/`-Werror` also lets the three existing `-Wunused-result` warnings (finding 237) accumulate silently. **Fixed:** `run_tests` now compiles with `-Wall -Wextra` always (which immediately surfaced ~50 previously-silent warnings, finding 237's among them); `-Werror` is deliberately left off until those are cleared. Added an opt-in `HS_SANITIZE` option (default OFF, so the pre-commit gate and default `cmake --preset tests` stay fast): on Linux/full-LLVM it enables ASan + UBSan with `-fno-sanitize-recover`; on the Windows emsdk clang (which ships no ASan runtime) it falls back to runtime-free UBSan trap mode, consistent with the engine's fail-fast doctrine. Enabling it immediately caught real UB — `signed-integer-overflow` in vendored `FastNoiseLite.h`'s wrapping hash arithmetic — now narrowly suppressed via `tests/ubsan-ignorelist.txt` (that one check, that one file only), leaving the sanitized suite green with full coverage of first-party code. Verified: default build passes the suite; `-DHS_SANITIZE=ON` builds, links, and runs green on the Windows toolchain. ## 7. Overall Impression & Appraisal of Merit

*This assessment is formed independently of the prior review.*

### Impression

Holosphere is what happens when a constraint-driven embedded mindset and a generative-art ambition are held by the same person long enough to stop fighting each other. The medium's constraints were not worked around; they were promoted into architecture. No debugger on a spinning rotor became an always-on trap doctrine with a death harness that *proves the traps fire*. A fragmenting heap became a partitioned arena whose every allocation site states its budget in bytes — and whose effects' comments do the arithmetic in public. No hardware on the desk became a WASM twin that is bit-faithful enough for the test suite to assert byte-exact determinism across the entire effect roster, and a four-board sync protocol whose failure modes were explored in a host simulator before a single Teensy spun. The result reads less like a hobby repository than like a small, opinionated rendering middleware that happens to ship on a 600 MHz microcontroller.

Two qualities dominate the reading experience. The first is the **comment culture**: across ~62k lines, non-obvious decisions carry their rationale, their failure mode, and frequently a measured error bound or a post-mortem of the bug that taught the lesson. Twelve reviewers on this pass — examining different components, unaware of each other — independently described their files as among the best-documented code of its scale they had assessed. The second is **consistency of doctrine**: fail-fast on cold seams, stripped asserts on hot paths, soft-degrade only for genuine transients, single-writer concurrency with written justification for every relaxed atomic. The doctrine is so uniformly applied that this audit's most common finding pattern is *the codebase violating its own standards in one place where it honors them in ten* — which is both a finding and a compliment.

The honest counterweight is unchanged from prior audits and is structural: this is a single-author system whose quality tracks the reach of its verification net. Host-testable C++ is superb and keeps getting better; the JS app outside the worker pool is good-but-unverified; the geometry tools are visibly the hobby corner of the hobby project. The risk model writes itself, and §5's findings distribute accordingly.

### The Achievement

Technically, the achievement is **vertical integration at a sustained craft bar that solo projects essentially never sustain**: one C++20 engine rendering Hopf fibrations, two reaction-diffusion systems on a precomputed Fibonacci-lattice graph, Hankin-method Islamic star patterns over a 52-solid Conway registry, sphere-traced volumetrics with analytic Lipschitz bounds, and OKLCH-perceptual palette animation — in 335 KB of deterministic memory with zero steady-state heap allocation, compiled unchanged into Teensy firmware, a four-MCU segmented driver with a clock-disciplined 1-wire sync protocol, a browser simulator, and a native test binary. The individual techniques are known; the conjunction, held to one standard, with the verification culture to prove it, is rare at any scale. The frame-sync layer deserves particular mention: a flywheel timebase that makes counter wrap structurally unobservable, a symbol alphabet designed so corruption degrades to *missed* rather than *wrong*, an emitter that self-censors rather than poison downstream phase — engineered, specified, simulated, and host-tested to a standard that would be respectable in commercial motor-control firmware.

Artistically, the effects are mathematics rendered honestly rather than textures wrapped on a ball: a genuine S³→S² fibration tumbling in 4D, real star-polygon intersections swept by the Hankin angle, cyclic competition producing rotating spiral waves on a true spherical lattice. The 16-bit linear pipeline and OKLCH interpolation — the refusal of the 8-bit sRGB shortcut the entire LED-art genre is built on — pay off in exactly the way audiences feel but cannot name: gradients that do not go muddy, hue walks that do not detour through brown. The live simulator makes the whole catalogue inspectable by anyone with a browser, which is itself a statement about confidence in the work.

### Standing Against Peers

**Against the hobby/maker LED ecosystem** (FastLED projects, WLED, one-off POV globes): categorically above, not marginally. POV spheres exist in number; a POV sphere with a unit-tested geometry kernel, death-tested invariants, a deterministic browser twin, a simulated multi-MCU sync protocol, CI-gated deploys, and a README that survives adversarial fact-checking is, to this audit's knowledge, unique. The trade is the usual one — a bespoke instrument, not a platform; no community, one product, bus factor of one.

**Against professional creative-technology offerings** (TouchDesigner, Notch, Pixelblaze, commercial LED controllers): the engineering *practices* — host-testable hardware seams, provenance-stamped artifacts, reflection-driven GUI generation, byte-exact determinism harnesses — meet or exceed what ships in small commercial tools, and the death-testing of the trap net exceeds what most professional teams build at all. What a commercial product would add is breadth-by-exposure: thousands of users finding the interaction seams this audit found by reading (the deep-link composition bug), and a hardware-in-the-loop rig to close the device shell that remains this project's only unverifiable territory.

**Against academic research:** the project applies known mathematics and would not claim a novelty bar at SIGGRAPH. As *artifact engineering*, however, it inverts the field's chronic weakness — where the median paper artifact is correct in principle and unrunnable in practice, this is integrative in principle and reproducible to the byte in practice. The spherical SDF/scanline rasterizer with analytic interval culling, the reaction-diffusion-on-Fibonacci-lattice implementation, and above all the frame-sync specification (quantitative drift budgets, failure matrices, assumptions made load-bearing and then simulated) are each executed at a quality that would embarrass a fair fraction of published systems work.

**Against the state of the art:** industrial POV displays with custom silicon and GPU-side spherical rendering define frontiers this project does not contest. Its claim is narrower and, as far as this audit can determine, real: it is among the most rigorously verified open implementations of the full conjunction — perceptually-correct 16-bit linear rendering, compile-time-specialized, formally fail-fast, bit-identical across microcontroller and browser, on a multi-MCU persistence-of-vision sphere. After this audit cycle, the claim is stronger than it was: the previous gap between "beautifully documented" and "proven" in the hardware layer has been closed everywhere a host can reach.

**Bottom line:** A− as engineering, for the second audit running — but a different A−. Last cycle the grade was held down by a P0 and an unimplemented protocol; this cycle it is held by a JS test gap, four unbaked palettes, and a dropped black frame. The work is converging on its own standard, and that standard is one most production codebases do not meet.
