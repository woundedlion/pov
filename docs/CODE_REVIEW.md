# Holosphere & Daydream — Comprehensive Code Quality Review

**Date:** 2026-06-11 (eighth audit; succeeds the seventh audit at tree `9e5a664`, whose findings 147–268 are all annotated resolved)
**Trees:** Holosphere `11532af`, daydream `3c22342`
**Scope:** Both repositories — the Holosphere C++ engine/firmware (`core/`, `effects/`, `hardware/`, `targets/`, `tests/`, build & CI) and the daydream web simulator (app modules, segmented-POV workers, geometry tools, JS tests, deploy). Out of scope by instruction: `core/effects_legacy.h`, `targets/Holosphere/Holosphere.ino`, `core/rotate.h`. `core/FastNoiseLite.h` is vendored third-party code, assessed only for integration.
**Method:** Fourteen independent component reviewers, each reading every in-scope file in full with the README as shared architectural context, grading objective quality (correctness, UB, races, bounds, performance, testability) and subjective quality (architectural elegance, interface expressiveness, naming, idiom). The prior `CODE_REVIEW.md` was deliberately not consulted during examination; it was reconciled afterward (§4), which removed nine findings (eight that existed only against a stale, since-cleaned worktree state, and one that re-raised a maintainer-rejected item) and demoted one. Finding numbers continue the project's running sequence: this audit's findings are **269–368** — 0 critical, 5 high, 19 medium, 76 low.

## 1. Executive Summary

The codebase remains at the high standard the audit series has documented, and the residual-defect profile continues to shrink in consequence: nothing found this pass corrupts a shipping default render path in the C++ engine. But unlike the seventh audit, this pass reports **five high-severity findings** — and their distribution is the most instructive result of the audit. Every one of them sits at a *verification boundary*: a formal ISR data race in the one foreground-polling loop the host suite structurally cannot execute (269); a semantic flaw in the one CSG operator the test suite never instantiates (270); a safety net (sanitizers) that a comment claims runs in CI but no workflow enables (271); and two web-tool defects — a styling 404 and a π-factor export mismatch — that exist only on the deployed site and in the export-paste workflow, which no automated test reaches (272, 273). Where the tests can see, the code is excellent; exactly where they cannot, the five worst residuals live. The codebase's quality gradient is now a near-perfect map of its verification reach.

The grade-limiting residuals cluster in three places:

1. **The device shell and exposed-but-unexercised engine API.** The `build_word_` handoff is the textbook missing-volatile ISR-flag bug, in a driver where every sibling handoff word is volatile and annotated (269). `SDF::Subtract` misreads its child's interval-fallback protocol and can silently erase the minuend — latent because no shipped effect composes Subtract, which is also why the suite's Union-only CSG coverage never caught it (270). Both are one-line fixes; both survived seven audits because nothing executes them.
2. **The daydream tools and their untestable inline logic.** All five tool pages serve unstyled on the live GitHub Pages deploy (272), the lissajous designer exports a phase the engine misinterprets by a factor of π (273), the palette tool's procedural export is not the C++ initializer the README promises (291), and ~3,500 lines of pure exporter/math logic sit in inline HTML scripts that the Node suite cannot import (292) — which is precisely where every fidelity bug found this pass lives.
3. **Verification gaps the project has already named.** The sanitizer job the test comments promise (271), the 96×20 device resolution that never runs under native asserts (286), and the MeshOps/spline WASM bindings with zero runtime CI coverage (285) are all gaps the codebase's own comments or prior audits identify; they remain open.

The deepest structural observation repeats from the prior audit, with sharper evidence: the doctrine — fail-fast cold seams, stripped-assert hot paths, byte-documented arena budgets, wrapped trig phases, baked palettes — is now so uniformly applied that most findings are *internal inconsistencies with the project's own standards*: the one unbaked palette among siblings that bake (282), the one unwrapped time source in a component that wraps three others (320), the one handoff word that isn't volatile (269), the one stale-threshold ordering among effects that order correctly (281). A separate cross-validation fell out of this audit by accident: at examination time the worktree carried stale reverts of roughly fifteen prior fixes, and the reviewers — reading cold — independently re-discovered eight of the reverted defects before the state was cleaned (§4). The audit series is converging and self-consistent.

**Verdict: A− overall.** The math/color core, the hardware protocol layer, and the C++ test suite each grade A on their own. The distance to a flat A consists of the five boundary-dwelling highs, the tools tier, and the named verification gaps.

## 3. Component Assessments

**core: math & geometry** (`3dmath.h`, `geometry.h`, `util.h`, `easing.h`, `waves.h`, `constants.h`, `static_circular_buffer.h`, `concepts.h`). Exceptionally high-quality foundation. The quaternion rotate expansion, Hamilton product, Marsaglia sampling, and the Bhaskara/Abramowitz-Stegun/Halley fast approximations (each with measured error bounds) all verify against canonical formulations. The degenerate-input policy — strict trapping `normalized()` for logic bugs vs `normalized_or()` for legitimate geometric edges — and the three mutually consistent `STEREO_INF` sentinel thresholds are exemplary design with written rationale. All residual defects here are low severity.

**core: color** (`color.h`, `palettes.h`, `color_luts.h`, `styles.h`, `generate_luts.py`). The 16-bit linear pipeline is provably correct: the `lerp16` round-to-nearest identity is exact at both endpoints (independently verified), the OKLab matrices match Ottosson's reference, and the LUT generator reproduces the checked-in tables bit-for-bit. `StaticPalette`'s two-axis compile-time modifier composition is genuinely expressive. Main residual: `GenerativePalette::get` re-converts its stops sRGB→OKLCH on every call, which a full-screen 4× SSAA shader pays per subsample (274).

**core: render** (`canvas.h`, `filter.h`, `sdf.h`, `scan.h`, `plot.h`, `transformers.h`). The variadic Pipeline's compile-time domain conversion, the double-buffer gate (with a written non-race proof and a watchdog), and the systematic theta-seam/pole handling are all verifiably correct. The one high-severity engine defect lives here: `SDF::Subtract` misreads a child's "cannot compute intervals" fallback as full-row coverage and can silently erase the minuend (270) — latent, but a serious flaw in an exposed CSG API whose sibling operators handle the same case correctly.

**core: animation & registry** (`animation.h`, `presets.h`, `effect_registry.h`, `effects.h`, `effects_engine.h`, `platform.h`, `led.h`). The Timeline's inline type-erased event storage with a pinned-handle trap, compile-time borrow contracts via deleted `const&&` overloads, and the LTO-proofed self-registering factory are carefully reasoned and test-pinned. `HS_CHECK` itself is a textbook fail-fast trap, death-harness verified. Residuals: a latent `cancel()`-on-repeating-animation zombie (283) and several non-compiling README examples (335).

**core: memory & mesh** (`memory.h/.cpp`, `mesh.h`, `conway.h`, `hankin.h`, `solids.h`, `spatial.h`, `generators.h`, `reaction_graph.h/.cpp`). Arena bounds math is overflow-proof by construction; debug builds have two-layer lifetime detection; the Conway operator scaffolds eliminate the copy-paste that usually plagues such code; load-bearing contracts (scratch split, composition polarity) are documented at the exact refactor site with do-not-standardize warnings. The notable residual is `ArenaVector::bind`'s debug/release semantic divergence on stale bindings (279).

**effects** (28 headers). Discipline is uniform: init-time persistent allocation with byte-documented budgets that check out exactly, ScratchScope'd per-frame state, zero heap, baked-palette LUTs on hot paths, `markAnimated`/pause-aware parameter wiring. Genuine algorithmic depth: correct real spherical harmonics with Condon-Shortley phase and provably exact (l,m) decoding, an exact trig-free double-angle lens in Liquid2D, the Hopf parameterization with 4π-aware phase wrapping, KD-tree-accelerated Voronoi with an exactness proof. Residuals concentrate in slider-extreme regimes (281), one device-build compile hazard (280), and perf inconsistencies between sibling effects (282, 311).

**hardware** (`dma_led.h`, `hd107s_frame.h`, `pov_segment_map.h`, `pov_single_map.h`, `pov_single.h`, `pov_sync.h`, `pov_segmented.h` + sync spec). The strongest component. Every load-bearing decision (flywheel math, symbol codec, acceptance gate, epoch scheduling, strip index math, wire format) lives in host-testable pure code under thin device shells; the 1,128-line spec matches the code constant-for-constant; the 1,520-line sync test includes a 4-board simulator with crystal skew, EMI, drops, and reboots. The one high-severity C++ defect is here, in exactly the territory no host test reaches: `build_word_` is the single ISR-to-foreground handoff that is neither volatile nor atomic (269).

**targets / build / CI** (`wasm.cpp`, `param_marshal.h`, `Phantasm.ino`, CMake, workflows, scripts, hook). The WASM bridge neutralizes every classic embind hazard: pre-sized never-resized buffers against `ALLOW_MEMORY_GROWTH` view detachment, transactional setters, validated untrusted inputs, generation-stamped tooling wrappers. Build flags show deep care (`-fno-finite-math-only` ordering with documented LTO-safe rationale). The gap is coverage, not code: the claimed sanitizer job runs nowhere (271), and the MeshOps/spline bindings have zero runtime CI coverage (285).

**tests** (C++ suite). ~15.3k lines across 34 modules that assert real invariants with independent oracles: brute-force KD-tree comparison, Euler-characteristic checks across operators and seeds, a conservative-cull oracle for the rasterizer, double-precision references for fixed-point math, and a death harness that proves all 26 trap sites fire with the exact signal asserted. Cross-run byte-identical determinism under an injected mock clock replaces brittle golden frames. Gaps: no native pass at 96×20 (286), no test filter on a single-TU monolith (346), `-Werror` still off (345).

**daydream: core app** (`daydream.js`, `driver.js`, `state.js`, `gui.js`, `sidebar.js`, `recorder.js`, `index.html`, importmap). Modern ES-module code with a well-engineered WASM lifecycle (spec-correct detach healing, fail-loud alias invariant, explicit degrade-vs-trap doctrine) and on-demand rendering behind a fixed-timestep accumulator. Residuals are interaction-level: a global Space-key pause that fires during documented keyboard UI flows (287), a picture-in-picture view that duplicates the main camera against its documented purpose (288), and dead code retaining ~10 MB (348).

**daydream: segment workers** (`segment_controller.js`, `segment_worker.js`, `segment_layout.js`, `worker_protocol.js`). Defensively engineered: airtight generation fencing, correct transferable ownership in both directions, a serialized worker queue that closes the init race, and an exact-once tiling proof in tests. One medium residual: an init-phase worker fault (WASM fetch/instantiate failure) can never reach the visible fault overlay because the ready guard precedes the faulted check (290).

**daydream: tools, JS tests, deploy** (5 tool pages, shared helpers, 7 node:test suites, `deploy.yml`). The weakest slice. The deploy gate itself is exemplary (SHA-pinned engine checkout, hash-bound artifact trio with a dirty-tree guard), `color.js` mirrors the engine's OKLab matrices digit-for-digit, and the 43-test node suite is green and well-aimed. But the live GitHub Pages deploy serves all five tools unstyled (272), the lissajous export's phase convention is π× off from the engine's (273), and ~3,500 lines of pure logic sit in inline HTML scripts Node cannot import (292) — which is exactly where this pass's fidelity bugs were found.

## 4. Reconciliation with the Prior Review

*The grades and findings above were formed without reading the previous `CODE_REVIEW.md` (the seventh audit, findings 147–268). This section consults it after the fact.*

**A stale-worktree incident produced an accidental cross-validation.** At examination time, the working tree carried uncommitted reverts of roughly fifteen of the seventh audit's recorded fixes (restored `bitruncate`, the KDTree build-scratch leak, the HD107S 255/256 unity loss, the float-demoting test harness, dead test code, dead mesh includes, the reverted IntervalTimer/`age_prior`/hue-cache hardening, among others). The fourteen reviewers, reading cold, **independently re-discovered eight of the reverted defects** — in several cases with the same analysis and the same recommended fix as the original findings (206–209, 229, 245, 246). The maintainer has since identified the state as a stale worktree and restored HEAD, where all prior fixes verifiably exist (each was re-checked in the clean tree before this report was finalized); those eight findings are therefore *excluded* from §5. The incident is reported here because it is evidence of a rare kind: two independent audits, a year of fixes apart, converging on identical defects from identical code — the audit methodology reproduces.

**Maintainer rejections honored.** One examiner finding re-raising the rejected Dynamo `pull(leader)` dead-generality item (prior finding 116, rejected; re-raised and dropped once before in the seventh audit) was dropped from this report. Nothing below re-litigates the double-buffer design or the other standing dismissals.

**Residuals and refinements of prior fixes.** Several findings below are the next layer beneath a prior fix, not a contradiction of it: 328 (deep_tween's *all*-collapsed trail) is the sibling case of prior 197's fix, whose regression test covers only the newest-collapsed case; 277 (`fn_point`'s parameterization divergence) is the layer beneath prior 222, which correctly made Thrusters pass `params.radius` to a helper that itself disagrees with the renderer; 274 extends the prior audit's palette-baking campaign (155/156/220/221) to the remaining direct consumer and to the bake itself; 295 refines prior 177's fix choice (the added guard is an always-on `HS_CHECK` in a per-pixel path, where the doctrine — and the existing per-draw cold trap — call for a stripped assert); and 271 elevates what prior 160's fix note already named as "a natural follow-up": the sanitizer CI job, now urgent because the in-tree comment affirmatively claims it exists.

**Two assessments of the prior audit are contradicted by this pass.** (1) The seventh audit stated the "CSG interval algebra … survived adversarial tracing"; finding 270 shows `SDF::Subtract`'s fallback semantics are wrong, and the reason both audits' *testing* could not catch it is the same reason it survived: only `Union` is ever instantiated in tests. (2) The seventh audit graded ISR concurrency A on the basis that "every relaxed-atomic/volatile/plain choice carries a written single-core rationale"; finding 269 identifies the one handoff (`build_word_`) whose written rationale — "foreground reads are single aligned words" — addresses tearing but not compiler caching, and is therefore not a valid justification. Both contradictions land in precisely the territory the prior audit's own risk model flagged: "anything that cannot be asserted in CI should be presumed one notch lower."

**Trajectory across audits.** The seventh audit's verdict — that the remaining distance to a flat A was "test the deep-link layer, bake four palettes, and re-submit a dropped black frame" — was honest for what that audit could see, and those three items did land. This pass's five highs replace them with a sharper formulation of the same structural truth: the project's residual risk no longer lives in any code its verification net covers. It lives in the net's holes — the device-shell polling loop, the untested CSG operator, the unstyled deployed tools, the unrun sanitizer job. The overall grade stays A− for the third consecutive audit, but each audit's A− has been held by a strictly smaller and more peripheral class of defect.

### Critical

### High

269. ✅ **`build_word_` ISR→foreground handoff is neither volatile nor atomic; a downstream board's pre-join polling loop can legally be optimized into a permanent dark spin** — `hardware/pov_sync.h:1120` (written at `pov_sync.h:1096`, polled at `hardware/pov_segmented.h:285`)
   **Fixed:** `build_word_` is now `volatile uint32_t`, matching the five sibling ISR/foreground handoff words (`pending_effect_`, `pending_gen_`, `consumed_gen_`, `release_req_`, `release_ack_`) — the foreground now re-loads it each poll and observes the ISR's first publish instead of hoisting a stale zero. Spec §8.5 amended to state the `volatile` rationale (observe-the-publish, not merely tear-free aligned reads). Single aligned-word load on Cortex-M7, so zero perf cost. Native suite green.
   Every other ISR/foreground handoff word in the driver is volatile (`pending_effect_`, `pending_gen_`, `consumed_gen_`, `release_req_`, `release_ack_`), but `build_word_` is a plain `uint32_t`, and the comment defending it ("foreground reads are single aligned words") addresses tearing, not compiler caching. On a downstream board before its first build — the real boot path — the `run_show` loop body contains *zero* volatile accesses and zero opaque calls: `cur && consumed_gen_ == built_gen` short-circuits on the provably-null local `cur`, and the debug branch short-circuits on non-volatile `hs::debug` before reaching `millis()`. The entire loop is side-effect-free non-volatile loads, which the compiler may hoist (and under `[intro.progress]` may assume terminates) — the board never observes the ISR's `publish_build` and stays dark forever. Teensy builds default to -O2, where GCC routinely performs exactly this hoist. Formally a data race (UB); the textbook missing-volatile ISR-flag bug, in a codebase that got every neighboring handoff right. Fix: make it `std::atomic<uint32_t>` (relaxed) to match the siblings, and amend spec §8.5, which encodes the same flawed justification.

270. ✅ **`SDF::Subtract` treats B's full-scan fallback as full-row coverage, silently erasing A** — `core/sdf.h:782-789`
   **Fixed:** when B returns false (`!has_b`) Subtract now returns false too, requesting a full-row scan — matching Union/SmoothUnion — so `scan_region` evaluates `distance() = max(A, −B)` per pixel (correct) instead of skipping the row. Corrected the misleading `MockFullWidthShape` comment in the test harness (false means "cannot produce intervals", not "covers the row") and converted `test_subtract_full_width_b_yields_empty` → `test_subtract_full_width_b_requests_full_row_scan`, which now asserts the false (full-row-scan) return rather than the old true+empty behavior that encoded the bug. The full-row scan is the only correct path when B has no intervals, so no working path loses performance. Native suite green.
   `get_horizontal_intervals` returning false means "cannot compute intervals — caller must scan the full row," not "shape covers the full row." Subtract conflates the two: when B falls back (`!has_b`) it returns true while emitting nothing, so the row is skipped and A's geometry is dropped. Real children trigger it: `SDF::Ring` returns false for *every* row whenever its axis is near-vertical (`r_val < MIN_HORIZONTAL_PROJ`), and `SDF::AngularRepeat` returns false unconditionally — so `Subtract<A, Ring-near-pole>` renders nothing, and an animated B whose axis sweeps past vertical makes A blink out for those frames. The sibling Union/SmoothUnion handle the same case correctly by returning false; the conservative fix is the same one-line change (the per-pixel `max(A, −B)` distance remains correct, costing only perf). Latent — no shipped effect composes Subtract, and the test suite instantiates only Union — but a serious semantic flaw in an exposed CSG API.

271. **Sanitizers never run in CI despite the in-tree comment claiming the Linux CI job enables them** — `.github/workflows/ci.yml` / `tests/CMakeLists.txt:59`
   `tests/CMakeLists.txt` documents `HS_SANITIZE` with "enable with -DHS_SANITIZE=ON (the Linux CI job runs it)" and names ASan/UBSan the "only automated cover for the arena-lifetime hazards the death harness can't reach" — but no workflow passes `-DHS_SANITIZE=ON` (repo-wide grep: zero workflow references). The safety net described as the sole automated coverage for arena-lifetime UB runs nowhere automatically, and the comment actively misleads a maintainer into believing it does. The prior audit's fix note for finding 160 explicitly deferred this as "a natural follow-up"; it is now the oldest known-open gap in the safety story. Add the sanitized Linux job (ccache absorbs the cost); at absolute minimum correct the comment.

272. **All five tool pages serve unstyled on the live GitHub Pages deploy: `vendor/tailwindcss.js` is git-ignored with no CDN fallback** — `daydream/tools/lissajous.html:12` (same in mobius/palettes/solids/splines)
   Every tool page loads `<script src="../vendor/tailwindcss.js">`, but `/vendor/` is excluded by `.gitignore` and therefore absent from the Pages artifact. Confirmed live: the Tailwind URL returns 404 while the tool pages return 200 — so the published site serves all five tools with their utility-class layout missing (palettes' entire grid, mobius/solids control typography), and `palettes.html`'s `@import` of `../vendor/fonts/fonts.css` 404s too. Unlike three.js/lil-gui, which `vendor-importmap.js` resolves to the CDN, Tailwind has no fallback path; the README's claim that the tools "resolve from the CDN by default" holds only for the importmap libraries.

273. **The lissajous designer exports phase A in radians, but the engine interprets `a` as a multiple of π — exported curves don't match the preview** — `daydream/tools/lissajous.html:339`
   The tool's preview computes `cos(m1·t − a)` with A in raw radians (slider labelled "Phase Shift A (radians)", range 0..2π) and exports that raw value; the engine's `lissajous()` (`core/geometry.h:628`) computes `cosf(m1·t − a·PI_F)` — `a` is a phase-shift *multiplier*. Pasted into the real consumers (`ChaoticStrings`'s `LissajousParams`, `Comets`), the rendered curve matches the designer's preview only when A=0. The comment at lissajous.html:133 explicitly (and wrongly) claims preview/engine parity. Export `A/π` (or adopt the multiplier convention in the tool) and fix the comment.

### Medium

274. ✅ **`GenerativePalette::get` re-converts its stops sRGB→OKLCH on every call; a full-screen SSAA shader pays it per subsample** — `core/color.h:795`
   **Fixed:** `update_luts()` now caches each stop's OKLCH form in a `colors_oklch` array, and `get()` interpolates those directly via `oklch_to_pixel(lerp_oklch(...))` — dropping the 6 `powf` + 6 `cbrtf` + 2 `atan2f` per sample (and per 256-entry bake). The cache populates on every path that mutates the stops (construction, `lerp()`), both of which route through `update_luts()`. The computation is bit-identical to the old `lerp_oklch(CPixel, CPixel, p)` (it just hoists the conversion), so no output changes; native suite green.
   `get()` ends in `lerp_oklch` on CPixel stops, costing ~6 `powf` + 6 `cbrtf` + 2 `atan2f` per sample, though the stops change only at construction and in `lerp()`/`update_luts()`. GSReactionDiffusion calls `palette.get(t)` inside its full-screen 4×-SSAA shader — up to ~166k transcendental-heavy conversions per frame at 288×144. Caching the stops' OKLCH forms where they change reduces `get()` to a lerp plus the exact-cube inverse, and speeds every 256-entry bake as well.

275. ✅ **`StaticPalette::bind` traps a null source but silently stores null modifier pointers that `get()` dereferences per pixel** — `core/color.h:1221`
   **Fixed:** `bind()` now also `HS_CHECK`s the coord- and color-modifier chains via fold expressions (`((cms != nullptr) && ...)`), trapping a null modifier at the same cold wiring seam as the null source. Empty packs fold to `true`, so unmodified palettes are unaffected, and the per-pixel `get()` path is untouched — zero hot-path cost.
   A null modifier is the same wiring-bug class as a null source, and on Teensy 4.x a null-this member read does not fault (address 0 is readable), so the failure mode is silent garbage colors on the device — precisely what the bind-site `HS_CHECK` exists to prevent. A fold-expression check closes the gap at zero hot-path cost.

278. **`SDF::Warp::Twist::lipschitz` is below the true shear operator norm, so Raymarch's sphere tracing is not strictly conservative** — `core/sdf.h:2286-2291`
   The warp is a shear with `|g| = γ = n·A/s`, whose exact operator norm is `γ/2 + √(1+γ²/4)` — strictly above the `1 + γ/2` returned (~8% short at γ=1, ~21% at γ=2, ~41% at γ=4) — and the `max(s, R/2)` clamp understates γ further for points reachable inside the fast-path shell. The TU is wholly inside `#ifdef __EMSCRIPTEN__` and not part of the native test target; an embind signature typo sails through the local gate and fails only in the CI wasm job. Worth a comment at minimum, or an optional `emcc` syntax check when EMSDK is present.
343. **Unrestricted `push` + `pull_request` triggers double-run every PR-branch push** — `.github/workflows/ci.yml:6`. The concurrency groups key on refs that differ between the push and PR merge ref, so neither cancels the other; the Windows job is expensive. Restrict push to master.
344. **wasm-debug inherits the tight 8 KB `STACK_SIZE` with no smoke coverage at -O0** — `CMakeLists.txt:48`. -O0 can multiply frame sizes severalfold and neither CI nor the smoke script ever builds the debug preset; mitigated by `-sASSERTIONS=1` stack checking (loud abort, not silent corruption). Give debug a roomier stack or document the reliance.

**Tests**

345. **`-Werror` remains off for the test target although the warnings that justified deferring it have been fixed** — `tests/CMakeLists.txt:52`. The comment defers -Werror until the known warnings clear; they have (prior finding 237). Enabling it now makes suite hygiene self-enforcing — the fail-fast philosophy applied to the suite itself.
346. **No test selection: every invocation runs all ~34 modules, including 27 subprocess spawns and the double-render determinism pass** — `tests/run_tests.cpp`. Combined with the single-TU design, any one-line test edit recompiles everything and any debugging session pays for the full suite. A trivial argv-based module filter (mirroring the existing `HS_DEATH_CASE` env dispatch) would transform iteration speed at near-zero complexity.
347. **Harness stats counters are non-atomic but incremented from a second thread in the Canvas spin-wait test** — `tests/test_canvas.h:205-210`. Race-free today only by careful construction (the main thread is provably blocked inside the ctor spin while the helper asserts); the constraint is unstated, and a future `HS_EXPECT` before the `join()` would introduce a silent data race in the harness itself. Make the counters atomic or state the constraint at the harness level.

**Daydream: core app**

348. **Dead exports and never-read precompute state retain ~10 MB at Phantasm resolution** — `daydream/driver.js:93` (`pixelPositions`/`pixelMatrices`, rebuilt per resolution switch), plus exported-but-unreferenced `XY`, `prettify`, `coordsLabel` — the latter latently wrong (double radius scaling) if ever used.

The slider never takes focus (verified in lil-gui source), so protection rests entirely on the onChange auto-pause side effect; the comment overstates the guarantee. A pointerdown/up listener on the controller closes it.

A latent precedence inversion: a future programmatic `appState.set('resolution', …)` would be clobbered in the URL by the stale ad-hoc value at the next flush. 357. **Animations-paused state is not resynced to a freshly created worker pool** — `daydream/segment_controller.js:209`. The init message carries effect + param snapshot but not paused state, and the controller doesn't track it: re-creating the pool while paused (segment-count change, mode re-toggle) spawns workers that animate under a paused GUI. `InitMsg` has no `paused` field; the asymmetry with the carefully-handled param resync is the tell.

The engine uses `segment_map()`/`segment_x_col()`; the dangling cross-reference sends readers hunting for a nonexistent C++ symbol (and is mirrored into daydream's installed README copy).

## 6. Overall Impression & Appraisal of Merit

*This assessment is formed independently of the prior review.*

### Impression

The defining experience of auditing this codebase is running out of the usual things to find. Fourteen reviewers were instructed to hunt hard and manufacture nothing, across ~57k lines of C++ and ~12k lines of JavaScript, and between them produced **zero criticals** and a defect list dominated by latent edge cases, sibling-convention drift, and documentation mismatches — in a domain (bare-metal ISR firmware + template-heavy C++ + a hand-rolled WASM boundary) where codebases of this ambition usually bleed undefined behavior. The reason is visible on every page: the project does not merely follow good practices, it *encodes its practices as enforceable doctrine* — fail-fast traps that a death harness proves fire, borrow contracts as deleted overloads, arena budgets stated in bytes at the allocation site and verified by arithmetic in the comments, byte-exact determinism asserted across the entire effect roster, an X-macro roster that makes effect/test/registry drift a compile error. Most of what this audit found is the doctrine catching its own author: the one unwrapped phase, the one unbaked palette, the one non-volatile handoff word, the one CSG operator without a test. When a codebase's residual defects are best described as *exceptions to its own rules*, the rules have won.

The five high-severity findings sharpen rather than soften this picture, because all five live precisely at the edges of the verification net: a device-shell polling loop no host can execute, a CSG operator no test instantiates, a CI job a comment promises but no workflow runs, and a deployed-tools layer no automated check ever loads. The quality gradient of this project is, to an unusual degree of fidelity, a map of what its tests can reach — superb where they reach, merely good where they cannot. That is simultaneously a compliment (the methodology works) and the project's clearest roadmap.

The honest counterweight is structural, not behavioral: this is a single-author system, and the bottom tier of the quality gradient — the five geometry tool pages, with their inline scripts, hand-ported math, and export-fidelity bugs — is the corner where the author's attention visibly runs out. The B− there is a real grade about real code that real users load from the live site, today, unstyled (272).

### The Achievement

The technical achievement is **vertical integration held to a uniform craft bar**: a single C++20 engine that renders Hopf fibrations, two reaction-diffusion systems on a precomputed Fibonacci-lattice graph, Hankin-method Islamic star patterns over a 52-solid Conway registry, sphere-traced volumetrics with analytic Lipschitz bounds, and OKLCH-perceptual palette animation — inside 335 KB of deterministic arena memory with zero steady-state heap allocation — compiled unchanged into single-Teensy firmware, a four-MCU segmented driver, a browser simulator, and a native test binary. Any one layer would be a respectable project. The conjunction, with a 15k-line test suite whose oracles are independent of the code under test, is the rare thing.

Within that conjunction, two artifacts deserve singling out. The **Phantasm frame-sync layer** — a flywheel timebase that makes 32-bit counter wrap structurally unobservable, a count-coded symbol alphabet designed so corruption degrades to *missed* rather than *wrong*, an epoch/beacon content layer that lets a rebooted board rejoin the right effect fail-dark — is engineered, specified (1,128 lines that match the code constant-for-constant), and adversarially simulated (crystal skew, EMI, drops, reboots) to a standard that would be creditable in commercial motor-control firmware. And the **16-bit linear color pipeline** is not a slogan but a verified property: the fixed-point lerp's rounding identity is exact at both endpoints, the checked-in LUTs reproduce bit-for-bit from their generator of record, and the OKLab matrices match the published reference — the refusal of the 8-bit sRGB shortcut on which the entire LED-art genre is built, carried through to the SPI wire.

Artistically, the effects are mathematics rendered honestly: a true S³→S² fibration tumbling in 4D, real star-polygon intersections swept by the Hankin angle, cyclic competition producing spiral waves on a genuine spherical lattice, correct real spherical harmonics down to the Condon-Shortley phase. The aesthetic payoff of the engineering choices is the kind audiences feel without naming — gradients that stay luminous, hue transitions that never detour through mud — and the live browser simulator, byte-faithful to the firmware, is itself a statement of confidence: the whole catalogue is inspectable by anyone, at the real resolution, running the real code.

### Standing Against Peers

**Against the hobby/maker LED ecosystem** (FastLED projects, WLED builds, one-off POV globes): categorically above, not incrementally. POV spheres exist in quantity; a POV sphere whose geometry kernel is oracle-tested, whose invariant traps are death-tested, whose multi-MCU sync protocol was simulated before fabrication, and whose README survives adversarial fact-checking is — to this audit's knowledge — without peer. The corresponding trade: it is an instrument, not a platform; one artist, one product, bus factor of one.

**Against professional creative-coding tools** (TouchDesigner, Notch, Pixelblaze and commercial LED controllers): the engineering *practices* — host-testable hardware seams, provenance-verified deploys, reflection-driven GUI generation, whole-roster determinism harnesses — meet or exceed what ships in small commercial products, and the verified-trap discipline exceeds what most professional teams attempt. What the professional ecosystem has that this does not is exposure: thousands of users finding the interaction-level seams (the Space-key conflict, the PiP regression) that a solo project only finds by audit, and hardware-in-the-loop test rigs for the device-shell territory that remains this project's only structurally unverifiable code.

**Against academic work:** the mathematics is applied, not novel — no claim at the SIGGRAPH bar. As *artifact engineering* it inverts academia's chronic failure mode: where the median paper artifact is correct in principle and unrunnable in practice, this is integrative in principle and reproducible to the byte in practice. The spherical SDF rasterizer with analytic interval culling, the reaction-diffusion lattice, and above all the sync specification — quantitative drift budgets, a failure-mode matrix, every assumption made load-bearing and then simulated — are executed at a level a good systems-track program committee would respect.

**Against the state of the art:** industrial volumetric/POV displays with custom silicon define frontiers this project does not contest, and it should not be measured there. Its real claim is narrower and verifiable: among open, inspectable implementations of the full chain — perceptually correct 16-bit linear rendering, compile-time specialized, formally fail-fast, deterministic across microcontroller and browser, on a multi-MCU POV sphere — this is the most rigorously verified one this audit series is aware of. Eight audits in, the gap between what the project claims and what its tests prove has become the thinnest this reviewer has seen in a codebase of this scope.

**Bottom line: A− as engineering, for the third consecutive audit — each time held there by a smaller, more peripheral class of defect.** This cycle the grade is held by five findings at the verification frontier and a tools tier awaiting the discipline the rest of the system already enjoys. The trajectory is unambiguous: the project keeps absorbing its audits, and the standard it is converging on is one most production codebases never reach.
