## 1. Executive Summary

The codebase remains at the high standard the audit series has documented. Every fix recorded by the eighth audit (269–368) verifies present in the current tree, and on the default single-WASM-instance and single-Teensy render paths nothing this pass found corrupts a shipping frame. The grade stays **A−** — for the fourth consecutive audit — but this pass changes the *shape* of the residual-risk story in one consequential way.

**The headline finding is a safety-net gap the eighth audit missed: the engine's NaN-domain safety has no compile-time guard and is regression-tested only in a configuration that cannot fail (369).** The whole engine's pole/`acos`/`sqrt` domain safety — every `Spherical`, `vector_to_pixel`, `blend_alpha`, `Gradient::get`, and palette lookup that casts a float to an int — rests on `hs::clamp` mapping `NaN → hi` as a saturating guard before the cast. On the WASM release build (`-O3 -ffast-math -fno-finite-math-only`) that contract survives **only** because `-fno-finite-math-only` is re-applied after `-ffast-math`; `-ffast-math` alone implies `-ffinite-math-only`, which licenses the compiler to assume no NaN exists and fold the guard away. Five independent reviewers (platform, build/CI, color, math, tests) converged on the same conclusion: (a) the flag ordering is currently correct; (b) there is **no** `#error`/`static_assert` on `__FINITE_MATH_ONLY__`, so a future flag edit, a toolchain default change, or an `-ffinite-math-only` sub-target can silently re-break float→int domain safety engine-wide; and (c) the native test build uses **no** math flags, so the NaN-clamp tests (`test_blend_alpha_clamps_before_cast`, `test_gradient_get_clamps_out_of_range`, the Möbius-saturation test) all run under default IEEE semantics where the contract holds trivially — the at-risk codegen is never exercised. The eighth audit, with fourteen reviewers, recorded this area as "build flags show deep care" and graded it A. That a ninth pass found a systemic, unguarded dependency *underneath* a praised area is itself the most important result of this audit: the project's verification net is excellent where it reaches, and this is one more hole in it.

The grade-limiting residuals cluster, as in prior audits, at the **verification frontier**:

1. **The segmented render path.** The Phantasm four-board / daydream-worker path is exercised far less than the single-instance path, and two real defects live there: a `Pixel::Feedback` warp-field upsample that can read uninitialized scratch at non-downsample-aligned clip boundaries (370), and the segmented driver's effect-publication handoff resting on `volatile` + IRQ-off rather than a `std::atomic` release/acquire edge (395).
2. **Exposed-but-unexercised API and capacity coupling.** `Pipeline` does not `static_assert` the documented World-before-Screen filter ordering (371); `Dynamo`'s `World::Trails` capacity has no `static_assert` tying emission rate to the buffer bound at 288×144, where it can plausibly overflow (372); a device-budget animation-size guard is defeated by the test build's widened cap (373).
3. **Named or structural test gaps.** The `-ffast-math` config gap (369); the device `H_OFFSET=3` filter paths that still never run under native asserts even after the eighth audit added a 96×20 smoke pass (375); and rasterizer smoke assertions that check "something was drawn" rather than geometric correctness (405).

The deepest structural observation repeats and sharpens: the doctrine — fail-fast cold seams, stripped-assert hot paths, byte-documented arena budgets, wrapped trig phases, baked palettes — is so uniformly applied that most findings are *internal inconsistencies with the project's own standards*: the one accessor without the `num_frames-1` guard its siblings carry (378), the one composite that ignores its own kernel normalization (396), the one handoff that isn't atomic (395), the one filter ordering that isn't asserted (371). When a codebase's residual defects are best described as exceptions to its own rules, the rules have won.

**Verdict: A− overall.** The math/color/memory core, the hardware protocol layer, and the C++ test suite each grade A on their own. The distance to a flat A is the unguarded build-flag dependency (369), the segmented-path defects, the capacity/ordering `static_assert`s the engine elsewhere makes a habit of, and the tools tier.

## 3. Component Assessments

**core: math & geometry** (`3dmath.h`, `geometry.h`, `util.h`, `concepts.h`, `easing.h`, `waves.h`). Exceptionally well-documented foundation: every fast approximation (`fast_atan2`, `fast_acos`, `fast_sinf`, `fast_cbrt`) carries a measured error bound and a rationale for why it is used at that call site, and the degenerate-input policy (`normalized()` trap vs `normalized_or()`) is principled. The grade is held at B+ on correctness/tests by three real edges: a `fast_atan2` octant-diagonal discontinuity (cosmetic), a `Vector::slerp` that uses approximate `fast_sinf` in the divisor near θ→π where the quaternion overload deliberately uses exact `sinf` (377), and three `Orientation` zero-arg accessors that index `num_frames-1` without the guard their indexed siblings carry (378).

**core: color** (`color.h`, `palettes.h`, `styles.h`, `color_luts.h` [generated]). The 16-bit linear pipeline is verifiably correct: the `lerp16` round-to-nearest identity is exact at both endpoints, the OKLab matrices match Ottosson's reference, the shortest-arc hue interpolation is textbook, and the generated LUT reproduces bit-for-bit from `generate_luts.py`. `StaticPalette`'s two-axis compile-time modifier composition is genuinely expressive. Residuals: `Color4::operator+=` accumulates alpha unclamped past 1.0 (386), `operator*=` scales both color and alpha (premultiplied semantics) while `lerp` treats the type as straight-alpha (387), and the saturating-add path relies on the high lane being zero (correct but fragile, flagged for a comment).

**core: memory & containers** (`memory.h/.cpp`, `static_circular_buffer.h`, `inplace_function.h`, `generators.h`, `presets.h`). Arena offset/overflow math is exemplary — no-wrap subtractive bounds checks and a `SIZE_MAX/sizeof(T)` overflow guard rarely seen. Move semantics are correct (move-after-move safe), the dual-stamp dangling-span detection is clever, and the reentrant `generate()` depth counter is correctly placed. Residuals are low: an unguarded power-of-two-alignment assumption in `Arena::allocate` (379), an `inplace_function` move `noexcept` that would `std::terminate` on a throwing-move callable (latent), and a doc/behavior drift on `owns_live` after the eighth audit removed it from `bind`.

**core: render — sdf / scan / plot / filter / canvas.** The SDF shape/rasterizer split, the variadic `Pipeline`'s compile-time domain conversion, the double-buffer gate (with a written non-race proof and watchdog), and the systematic seam/pole handling are all verifiably correct. This tier holds the two highest-priority engine defects after the build gap: the `Pixel::Feedback` warp-field upsample can read uninitialized coarse-grid scratch at non-`ds`-aligned clip boundaries (370), and the documented World-before-Screen ordering is not `static_assert`ed so a mis-ordered pipeline silently round-trips coordinates (371). SDF residuals: a `Ring` fast-path that can blow up `half_width` near tangent/pole rows (380), `PlanarPolygon` tip-clipping at large radius (381), and a `Subtract`-of-a-stroke that can erase inter-edge columns (382 — a sub-case beneath the eighth audit's now-fixed 270). Scan residuals are the SSAA pattern (383/384) and a Volume pole-straddle under-scan (385).

**core: animation & transformers.** The Timeline's inline type-erased event storage with a pinned-handle trap, deleted-`const&&` borrow contracts, and the orientation-collapse pre-pass are carefully reasoned and test-pinned. Residuals: a `MobiusWarpEvolving` whose device inline-storage budget is checked only under the test build's *widened* cap (373 — High), `RandomWalk` chaining a `.then()` that can never fire (390), an unguarded `MeshMorph` vertex-count assumption (391), and a transformer pool-slot leak if a "repeating" spawn is `cancel()`ed (residual of the eighth audit's 283).

**core: mesh / conway / hankin / spatial / solids.** Arena polarity (primitive→`target`, composed→`temp`) is honored everywhere traced; half-edge linkage, snub Newell normals, the dual face/vertex swap, and the Hankin angle sweep are correct; the constexpr Platonic data was verified programmatically (manifold, consistently wound, outward normals, correct degrees), and the registry counts (16+13+23=52) and Collections spans are in bounds. The one inconsistency: a Hankin rosette index stored into an `int16_t` array without the `narrow_index` guard every sibling cast uses (392 — safe today only because of an upfront range check).

**core: platform / registry / led.** `HS_CHECK` is a textbook fail-fast trap (stdio-free on device, survives `NDEBUG`, death-harness verified); the Meyers-singleton registry defeats the static-init-order fiasco; `used+retain` defeats LTO dead-stripping; the deterministic `mt19937(1337)` seed is identical across targets. This component carries the audit's headline: the `hs::clamp` NaN→hi contract holds on every target as built, but for two *different* reasons (an x86 `minss` instruction quirk vs IEEE `fminf` NaN-suppression), the doc comment mis-states the second (376), and nothing guards the `-fno-finite-math-only` flag the WASM build depends on (369).

**hardware** (`dma_led.h`, `hd107s_frame.h`, `pov_segment_map.h`, `pov_single*.h`, `pov_sync.h`, `pov_segmented.h` + spec). The strongest component. The relaxed-atomic concurrency reasoning on the single-core Cortex-M7 is correct and rigorously justified in comments; the flywheel position math (64-bit, wrap-unobservable epoch fold), count-coded symbol codec, acceptance gate, and epoch lockstep were traced and hold; the host-testable split is exemplary. Residuals at the concurrency frontier: the `EdgeMailbox` `burst_complete()`/`claim()` split opens a narrow lost-first-edge window (374 — High), the segmented effect publication uses `volatile`+IRQ-off rather than a formal `std::atomic` release/acquire edge (395), and the `snap` gate doesn't re-verify boundary alternation on a same-boundary re-snap (393).

**targets / wasm bridge** (`wasm.cpp`, `param_marshal.h`, `Phantasm.ino`). The cleanest seam in the project. The never-resize pixel buffer (against `ALLOW_MEMORY_GROWTH` view detachment), the single-source param-marshal ordering, the X-macro resolution SSOT, the generation-stamped tooling arenas, and the validate-before-teardown setters all verified correct. Only Low items (a `size_t`→`int` narrowing in a dev-only binding; a stack-canary lower-bound caveat).

**tests** (C++ suite). ~15.3k lines across 34 modules with independent oracles (brute-force KD-tree, Euler characteristic, double-precision references, conservative-cull grids), a death harness asserting the exact trap signal, and byte-identical cross-run determinism under an injected mock clock. The eighth audit's sanitizer CI job and 96×20 smoke pass are both present. Remaining gaps are exactly the ones that define the residual-risk frontier: no `-ffast-math` build is tested (369), the `H_OFFSET=3` filter paths never run under asserts even at 96×20 (375), and the `scan`/`mesh_raster` smoke assertions verify drawing happened rather than where (405).

**daydream: core app & segment workers & tools.** Modern ES-module code with a well-engineered WASM lifecycle: the detach-heal path re-fetches `getPixels()` after every growth point and is verified against the vendored Three.js `bufferSubData`, the pixelToVector mapping matches the C++ engine exactly, and the fixed-timestep accumulator is spiral- and pause-safe. The segmented subsystem's generation fence is provably sound, transferable ownership is correct in both directions, and a real Node test suite (106 tests) covers the layout math, fence, and tool exporters. Residuals are interaction-level: a recorder that can capture cleared first frames in segmented mode (400), per-drag `window` listeners that can leak if the GUI is destroyed mid-drag (401), a camera-distance-dependent label-visibility threshold (402), and a `mapValue` divide-by-zero in the palette tool (403).

**Trajectory.** The eighth audit's distance-to-A was five boundary-dwelling highs and a tools tier; those are largely closed. The grade stays A− for the fourth consecutive pass, but the one item that would most change the project's risk profile is now identified.

### Critical

*None confirmed on the default single-instance render path — consistent with the eight prior audits. The single most important item, 369, is classified High below only because the offending flag is currently present and output is currently correct; multiple reviewers argued it is Critical-class by blast radius, and it should be treated as the top priority.*

### High

369. ✅ **The `hs::clamp` NaN→hi contract — on which engine-wide float→int domain safety depends — has no compile-time guard and is regression-tested only in a configuration that cannot fail** — `core/platform.h:~1083,1107` (contract), `CMakeLists.txt:62-70` (flag), `tests/CMakeLists.txt` (test config). The WASM release build is `-O3 -ffast-math -fno-finite-math-only`; `-ffast-math` implies `-ffinite-math-only` (assume-no-NaN), and the NaN-suppression that makes `clamp(NaN,lo,hi)==hi` survives **only** because `-fno-finite-math-only` is re-applied afterward. There is no `#error`/`static_assert` on `__FINITE_MATH_ONLY__`, so a flag reorder, a toolchain default change, or an `-ffinite-math-only` sub-target silently re-introduces UB float→int casts across `Spherical`, `vector_to_pixel`, `blend_alpha`, `Gradient::get`, and every palette lookup. The native test build sets **no** math flags, so the NaN-clamp tests validate the contract under default IEEE semantics — never the at-risk codegen. **Fix:** add `#if defined(__FAST_MATH__) && __FINITE_MATH_ONLY__ != 0` → `#error "hs::clamp NaN->hi contract requires -fno-finite-math-only after -ffast-math"` near the clamp definitions, and add a CI step (or test target) compiled with the real release math flags that runs the NaN-clamp assertions. **Resolved:** added a compile-time guard above the `hs::clamp` definitions in `core/platform.h` — `#if defined(__FINITE_MATH_ONLY__) && __FINITE_MATH_ONLY__ != 0` → `#error`, keyed on the actual breaking condition (finite-math assumed) rather than `__FAST_MATH__`, which clang *undefines* once `-fno-finite-math-only` peels a sub-option off `-ffast-math`. It fires on every TU that includes `platform.h`, so dropping `-fno-finite-math-only` from the WASM release line — or adding a bare `-ffast-math`/`-ffinite-math-only` sub-target — fails the build here instead of silently re-introducing the cast UB. Verified the guard fires under a bare `-ffast-math` syntax-check and passes under the real `-ffast-math -fno-finite-math-only`. Added the exercising target the finding asked for: `tests/fastmath_clamp_check.cpp` recompiles the existing clamp-before-cast / singularity-saturation assertions (`test_blend_alpha_clamps_before_cast`, `test_pixel16_scale_clamps_before_cast`, `test_gradient_get_clamps_out_of_range`, `test_mobius_longitude_singularity_saturates_to_endpoint`) under `-O3 -ffast-math -fno-finite-math-only` (the WASM release math flags), registered as CTest `unit_fastmath_clamp` and added to CI shard 2 so it is not orphaned — all assertions pass against the shipping codegen.
370. ✅ **`Filter::Pixel::Feedback` warp-field upsample can read uninitialized coarse-grid scratch at non-downsample-aligned clip boundaries** — `core/filter.h:~1262-1323`. Step 1 populates coarse rows `[cy_lo, cy_hi]` with `cy_hi` clamped down to `hh-1`; Step 2 samples `cy0=y/ds` and `cy1=min(cy0+1,hh-1)`. When the segment's `y_hi` is not a multiple of `ds`, a sampled pixel near the bottom of the band can read a coarse row in `(cy_hi, hh)` that Step 1 never wrote, bilerping garbage warp deltas (stale scratch — visual corruption, not a crash) on the Phantasm segmented / daydream-worker path. **Fix:** make Step 1's populated band a strict superset of Step 2's read set (`cy_hi = min(hh-1, ((y_hi-1)/ds)+1)`), and assert in Step 2 that every `cy1` read was populated. The filter reviewer rated this Critical; confirm against a segment whose height is not a multiple of `style.downsample`. **Resolved:** the band-coverage half was already in place — commit `067892ad` set `cy_hi = min(hh-1, ((y_hi-1)/ds)+1)`, the exact superset of the rows Step 2 reads (verified: for any `y_hi`, max read `cy1` equals `cy_hi`), so no uninitialized row is sampled. Added the recommended defensive guard: a per-row (not per-pixel) `HS_CHECK(cy0 >= cy_lo && cy1 <= cy_hi)` in Step 2 that traps if a future edit to either band formula lets a read escape the populated band. Verified via `filter`, `pov_segmented`, and `MeshFeedback` at the non-`ds`-aligned 96×20 segment size — all pass, guard never fires.
371. ✅ **`Pipeline` does not `static_assert` the documented World-before-Screen filter ordering** — `core/filter.h:~200-383`. A pipeline such as `Pipeline<W,H, Screen::AntiAlias<W,H>, World::Orient<W>>` compiles; at runtime the 2D AntiAlias splat is lifted back to a Vector via `pixel_to_vector`, rotated, and re-projected, destroying the AA distribution and doing the rotation in the wrong space — silently. Only the *terminal*-filter rule is asserted. **Fix:** add a compile-time scan asserting no `is_2d` Head precedes an `is_3d` Tail member, matching the existing terminal-filter assert pattern (and the user-memory namespace convention). **Resolved:** added a `static_assert(!Head::is_2d || (... && Tail::is_2d), ...)` beside the existing terminal-filter assert in `Pipeline<W,H,Head,Tail...>`. At every recursion node it requires that if this stage is 2D, no later stage is 3D; composed across the recursion that enforces the chain stays 2D once it goes 2D, i.e. all `World::*` (3D) stages precede every `Screen::*`/`Pixel::*` (2D) stage. Verified the four multi-stage shipping pipelines (SplineFlow, MobiusGrid, Dynamo, MeshFeedback) and the `FlowField`/`PetalFlow`/`Moire` World→Screen chains still compile and the full suite passes, and that the previously-silent `Pipeline<W,H, Screen::AntiAlias, World::Orient>` now fails to compile with the new message.
372. **`Dynamo`'s `World::Trails<W, 10000>` capacity has no `static_assert` tying emission rate to the buffer bound; it can plausibly overflow at 288×144** — `effects/Dynamo.h:~392` (consumed `:435`, flushed `:287`). ## 6. Overall Impression & Appraisal of Merit

*This assessment is formed independently of the prior review, from this audit's own twenty-two component examinations.*

### Impression

The dominant experience of reviewing this codebase is the steady erosion of a reviewer's usual ammunition. Twenty-two independent reviewers were told to hunt hard across ~33k lines of hand-written C++ and ~5k lines of JavaScript, in a domain — bare-metal ISR firmware, template-metaprogrammed rendering, a hand-rolled WASM/JS memory boundary, a multi-MCU wire protocol — where ambitious projects normally hemorrhage undefined behavior, data races, and buffer overruns. They returned **no defect that corrupts a shipping frame on the default render path**, and a list dominated by latent edge cases, sibling-convention drift, and documentation mismatches. The reason is visible on every page: this project does not merely *follow* good practice, it *encodes practice as enforceable machinery* — fail-fast traps a death harness proves fire with the exact signal, borrow contracts expressed as deleted overloads, arena budgets stated in bytes at the allocation site and checked by arithmetic in the comments, byte-identical determinism asserted across the entire effect roster under an injected clock, and an X-macro roster that turns effect/test/registry drift into a compile error. The most striking property is that the great majority of what this audit *did* find is the doctrine catching its own author: the one accessor missing a guard its siblings carry, the one composite ignoring its own normalization, the one phase left unwrapped, the one handoff that isn't atomic, the one filter ordering that isn't asserted. When a system's residual defects are best described as exceptions to its own rules, the rules are doing real work.

What gives this pass its edge over a simple re-confirmation is that the single most consequential finding does not fit that comfortable pattern. The engine's entire float→int domain-safety story — every `acos`, `sqrt`, and palette cast across the renderer — rests on one clamp behaving a certain way under NaN, which in turn rests on one compiler flag (`-fno-finite-math-only`) surviving in one CMake line, guarded by nothing and tested only in a configuration that cannot exercise the risk. That dependency sat unflagged through eight prior audits and a praised build tier. It is correct *today*. But it is the kind of thing that, the day it breaks, breaks silently and everywhere — and finding it is a reminder that even a verification net this dense has a topology, and the holes are exactly where the worst residuals live: the segmented render path, the unexercised CSG operator branch, the device-only filter offset, the build configuration the code's safety quietly assumes.

The honest counterweight is structural, not behavioral. This is a single-author system. The lowest tier of the quality gradient — the geometry tool pages — is where one person's attention visibly thins, and the deepest verification gaps are precisely the territory a solo project cannot cheaply cover: the device-shell ISR loops no host can run, the hardware-in-the-loop timing a unit test cannot measure, the interaction seams that thousands of users would surface in a week. The grade is held at A− by real code that real users touch, not by a rounding error.

### The Achievement

The achievement is **vertical integration sustained at a uniform craft bar across an unusually wide stack**. A single C++20 engine renders Hopf fibrations, two reaction-diffusion systems on a precomputed Fibonacci-lattice k-NN graph, Hankin-method Islamic star patterns over a 52-solid Conway-operator registry, sphere-traced volumetrics with analytically-derived Lipschitz bounds, and OKLCH-perceptual palette animation — inside 335 KB of deterministically partitioned arena memory with zero steady-state heap allocation — and compiles *unchanged* into single-Teensy firmware, a four-MCU segmented driver, a browser WASM simulator, and a native test binary. Any one of those layers would be a creditable standalone project. Their conjunction, wrapped in a ~15k-line test suite whose oracles (brute-force KD-tree, Euler characteristic, double-precision references, conservative-cull grids) are genuinely *independent* of the code under test, is the rare thing.

Two artifacts deserve singling out on their own merits. The **Phantasm frame-sync layer** is the standout: a flywheel timebase that makes the 32-bit cycle-counter's 7-second wrap structurally unobservable, a count-coded symbol alphabet deliberately designed (odd-only, distance-2) so a corrupted symbol degrades to *missed* rather than *wrong*, and an epoch/beacon content layer that lets a rebooted board rejoin the correct effect fail-dark rather than fail-wrong — specified in a datasheet that matches the implementation constant-for-constant and adversarially simulated against crystal skew, EMI, dropped symbols, and reboots. That is a standard one expects from commercial motor-control or avionics firmware, not an LED-art piece. And the **16-bit linear color pipeline** is not a marketing line but a verified property: the fixed-point lerp's rounding identity is exact at both endpoints, the committed LUTs reproduce bit-for-bit from their generator of record, the OKLab matrices match the published reference — the genre's universal 8-bit sRGB shortcut refused all the way to the SPI wire.

Artistically, the effects are mathematics rendered without cheating: a true S³→S² fibration tumbling in 4D, real star-polygon line intersections swept by the Hankin angle, cyclic-competition dynamics producing genuine spiral waves on a spherical lattice, correct real spherical harmonics down to the Condon-Shortley phase. The aesthetic payoff of the engineering — luminous gradients, hue transitions that never detour through mud — is the kind an audience feels without being able to name, and the browser simulator, byte-faithful to the firmware, makes the whole catalogue inspectable by anyone at the real resolution running the real code.

### Standing Against Peers

**Against the hobby/maker LED ecosystem** (FastLED/WLED builds, one-off POV globes): categorically, not incrementally, above. POV spheres exist in quantity; a POV sphere whose geometry kernel is oracle-tested, whose invariant traps are death-tested, whose multi-MCU sync protocol was simulated *before* fabrication, and whose README survives adversarial fact-checking is, to this audit's knowledge, without peer in that ecosystem. The trade is the usual one for a work of this kind: it is an instrument, not a platform — one artist, one product, bus factor of one.

**Against professional creative-coding tools** (TouchDesigner, Notch, Pixelblaze, commercial LED controllers): the engineering *practices* — host-testable hardware seams, provenance-verified deploys, reflection-driven GUI generation from the C++ parameter registry, whole-roster determinism harnesses — meet or exceed what ships in small commercial products, and the verified-trap discipline exceeds what most professional teams attempt. What the professional ecosystem holds that this does not is exposure: a user base large enough to find the interaction-level seams a solo project only finds by audit, and hardware-in-the-loop rigs for the device-shell code that remains structurally unverifiable here.

**Against academic work:** the mathematics is applied, not novel — no claim at the research frontier, and none is made. But as *artifact engineering* it inverts academia's chronic failure mode. Where the median paper artifact is correct in principle and unrunnable in practice, this is integrative in principle and reproducible to the byte in practice — and the sync specification in particular (quantitative drift budgets, an explicit failure-mode matrix, every assumption made load-bearing and then simulated) is executed at a level a systems-track program committee would respect.

**Against the state of the art:** industrial volumetric and POV displays built on custom silicon define frontiers this project does not contest and should not be measured against. Its real claim is narrower and verifiable: among *open, inspectable* implementations of the full chain — perceptually-correct 16-bit linear rendering, compile-time-specialized, formally fail-fast, deterministic across microcontroller and browser, on a multi-MCU POV sphere — this is the most rigorously verified one this reviewer is aware of.

**Bottom line: A− as engineering.** The math/color/memory core, the hardware protocol layer, and the test suite each stand at A on their own. The distance to a flat A is now a single, sharply-defined list: guard the build-flag dependency the whole renderer silently trusts (369), close the two segmented-path defects, add the capacity and filter-ordering `static_assert`s the engine already makes a habit of elsewhere, and bring the tool tier up to the standard the rest of the system enjoys. That is a remarkably short and peripheral list for a system this ambitious — which is the truest measure of the achievement.
