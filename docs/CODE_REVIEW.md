# Holosphere — Code Quality Review

*Fifth independent multi-agent audit, 2026-06-10. The codebase was partitioned into eighteen component groups spanning the full C++20 engine (`core/`, `effects/`, `hardware/`, the WASM bridge, the C++ test suite, the build/CI plumbing) and the sibling `daydream/` JavaScript simulator (renderer, GUI/recorder, segmented-POV worker pool, geometry tools). One reviewer read every file in its group against the project's intent; **89 candidate findings** were then each handed to a separate adversarial validator that re-opened the source and weighed it for **truth, relevance, and whether a fix is worth a developer's time**. **57 survived; 32 were dismissed.** A final synthesis pass aggregated the per-component grades and the surviving findings into the report below. Out of scope by request: `core/effects_legacy.h`, `targets/Holosphere/Holosphere.ino`, and `core/rotate.h`; `core/FastNoiseLite.h` is treated as vendored, and the generated tables (`core/color_luts.h`, `core/reaction_graph.cpp`) were spot-checked rather than read line by line. Letter grades (A+ … F) are measured against a senior / graphics-middleware professional bar — not "does it run."*

*This pass post-dates a fourth audit recorded earlier the same day, and its picture has been **verified to have changed underneath it**: every open P2/P3 recommendation that audit raised has since landed in the tree — BZ's perturbation now draws `rand_int(0, 3)` (species C reachable), `Pixel16::operator*` now clamps with the NaN-safe `hs::clamp`, `World::Trails` gained a `set_lifetime` setter, `to_short` now clamps in float space, and `Presets` carries `static_assert(Size > 0)` — and the §4a `ease_in_out_cubic` rename, the `vectorToLogPolar` south-pole sentinel, and the daydream label-helper HTML fix are committed. Two of this pass's findings (#16, #23) are in fact **direct consequences** of those very fixes. What is genuinely new here is breadth: this audit ran the daydream JS test suite (finding one red test the prior pass never executed), reached the WASM/JS bridge and the firmware frame-sync startup, and so located surviving debt at the **integration seams** the narrower passes did not exercise. The §§ "Reconciliation with the Prior Audit", "Overall Impression", "Technical & Artistic Merit", and "Comparative Appraisal" below are formed independently and deliberately do **not** reuse the prior audits' framing or conclusions.*

## Executive Summary

Holosphere is a genuinely high-caliber piece of systems engineering: a single C++20, compile-time-`<W,H>`-templated rendering engine that drives both a 480 RPM persistence-of-vision LED sphere (Teensy 4.x firmware) and a browser WASM simulator (`daydream`, Three.js). Judged at a senior/professional bar, the codebase is in the top tier for correctness discipline, numerical care, documentation density, and test rigor. The design is opinionated and coherent — arenas because the Teensy heap fragments, always-on `HS_CHECK` traps because there is no on-device debugger, exact FastLED mocks so the simulator predicts the device, and zero-runtime-generality compile-time specialization throughout.

The engine core (math, color, SDF/scan, plot/filter, animation, mesh, memory, hardware drivers) is excellent and largely defect-free; the confirmed bugs there are latent and low-impact. The real concrete defects cluster at the **seams** — the WASM/JS bridge, the segmented-POV worker pipeline, the firmware frame-sync startup, and effect parameter extremes — which is exactly where a system this internally-consistent tends to leak. The most actionable items: a WASM `fromData` missing-`UINT16_MAX`-bound that silently truncates indices and then aborts the whole module (high), a **currently-failing** `daydream` worker unit test on the most data-flow-critical path (high), a reachable timeline-event overflow in ShapeShifter that silently drops visuals (medium), a narrow-but-latching frame-sync parity-seed hazard on downstream boards (medium), and a screenshot-tooling effect list that has already drifted from the registry (medium).

The web layer is polished and careful for browser code (detached-buffer guards, aliased zero-copy display buffer with asserted invariants, thorough teardown), but its JS test coverage is thin outside `state.js` and the codec picker, and `palettes.html` is a maintainability outlier.

**Overall grade: A-**

### Correctness — A-
The engine math is correct and verified by behavior-focused, oracle-driven tests: stereo/gnomonic round-trips, quaternion identities, Euler-characteristic checks across four Platonic seeds, brute-force KD-tree cross-checks, and sorted-interval CSG regressions. Pole/antipode/singularity handling is systematic. The deductions are concrete and concentrated at boundaries: the WASM `fromData` 16-bit index gap (high), a failing worker compositor test (high), ShapeShifter's `66 > 64` timeline overflow (medium), the frame-sync parity-seed hazard (medium), plus latent low-impact bugs (AngularRepeat non-Y vertical bounds, Flower pole wraparound, `Color4::lerp` alpha/color clamp asymmetry, `BakedPalette::get` NaN UB, the Lissajous export argument-order mismatch).

### Architecture & Design — A
Compile-time `<W,H>` specialization, single-arena partitioning with explicit `Arena&` threading, the variadic `Pipeline<W,H,Filters...>` with `if-constexpr` domain transitions, the half-edge mesh core reduced to shared orbit/edge-walk primitives, and the X-macro single-source-of-truth effect roster form a tight, intentional whole. `daydream` mirrors it with single-writer URLSync, lazy-getter worker injection, and a one-frame segmented pipeline matching the multi-Teensy hardware model. Minor smells only: Liquid2D/Flyby skeleton duplication, the dead `setSegment` protocol path, and a slightly overloaded driver class.

### Interface Expressiveness & API Design — A-
Strong intent-revealing design (named factories, `normalized_or` split, deleted-rvalue borrow guards, typed owned-vs-borrowed spans, boolean-returning WASM setters). Held below A by under-specified or fragile surfaces (`inv_gnomonic` sign overload, per-shape `DistanceResult` field semantics) and a recurring dead-API theme across both repos.

### Code Readability & Maintainability — A-
Comments explain the *why* to an unusually high standard. The recurring drag is duplication and dead stores (`static inline` easing, dead pre-C++20 `lerp`, `to_short`/`float_to_pixel16` twins, `Mutation::from`, triplicated mesh test fixtures) and one genuine outlier: `palettes.html`, a 1500-line inline monolith littered with `NEW:`/`MODIFIED:`/`FIXED:` change-log comments that violate the project's own no-rename-history convention.

### Performance & Efficiency — A
Disciplined hot/cold separation, fixed-point and LUT-backed hot paths with measured error budgets, zero per-pixel allocation, single-DMA-transfer hardware path, zero-copy WASM/worker transfers. Minor: per-ring `make_basis` recompute, per-point Comets rasterization, and unbounded float phase accumulation that degrades trig precision on multi-hour runs.

### Memory Safety & Resource Management — A-
Wrap-safe arena math, two-axis use-after-free tracking, multiply-site overflow traps, RAII evacuators, trapping narrows, and diligent GPU/WASM disposal. The one concrete gap is the `fromData` silent `uint16` narrowing; latent sharp edges are `ArenaSpan`-from-temporary, `clone_from` null-source memcpy, and a resize-time material/uniform aliasing fragility in `driver.js`.

### Error Handling & Robustness — A-
The fail-fast doctrine is applied with precision and the untrusted JS/WASM boundary correctly rejects-rather-than-traps — *except* the one `fromData` hole. Other gaps: ShapeShifter relying on Timeline's silent drop at a reachable cap, `Trails` negative re-emitted age after `set_lifetime` shrinks, and `solids.html` rendering a partial mesh after a caught op error.

### Testing & Verification — A-
The C++ host suite alone would earn an A: analytic/oracle/invariant assertions, signature-pinned regressions, a death harness asserting the *exact* trap signal, and roster-generated coverage that cannot drift. The aggregate drops to A- for systemic gaps — a failing JS worker test, near-zero coverage of `driver.js`/compositor/geometry math, no runtime exercise of the shipped WASM module, untested `fromData` rejection paths, and several untested heavier animation types.

### Documentation — A
Load-bearing, accurate inline documentation across both repos, with cross-referenced contracts (WASM memory-view detachment mirrored in JS, provenance threat model, fast-math flag rationale). Isolated gaps: the H_OFFSET device/host divergence documented only in a test comment, `DistanceResult` per-shape semantics, a misplaced `read_id` comment, and the `palettes.html` comment noise.

### Portability & Build System — A-
Clean three-way platform split, correct and rare `-ffast-math`/`-fno-finite-math-only` handling, an exemplary WASM provenance trio gated on the recorded engine SHA, pinned toolchain/actions, and robust Windows-Clang detection. Deductions for the H_OFFSET parity gap, the ODR-fragile header-scope `instance_`, host divide-by-zero edges, and the silent `build_release.bat` install no-op.

### Architectural Elegance — A
Split-table LUT reconstruction, sentinel-unified projection algebra, shared SDF interval emitters, per-face Lipschitz-bounded LUTs, compile-time `StaticPalette` folds, `opaque()`-defeated death tests, self-referential determinism checks, and the self-locating build-time importmap are all tasteful. A few seams (global-constant sprawl, duplicated skeletons, dead protocol members) keep it just shy of flawless.

## Prioritized Fix List

### Critical

*(none — no defect rises to data-loss, security, or guaranteed-crash-in-normal-use severity; the firmware's fail-fast traps are intentional)*

### High

1. ✅ **WASM `fromData` omits `UINT16_MAX` vertex/index bound** — `targets/wasm/wasm.cpp:468-581` — a >65535-vertex/index editor mesh fits the 8 MB tooling arena, silently narrows `int`→`uint16_t` at the `m.faces.push_back(idx)` (line 572), corrupts topology, then aborts the whole module on the next operator via `build_from_flat`'s `HS_CHECK` — defeating the file's own reject-don't-trap intent. → After computing `num_verts`/`num_face_verts`, add `if (num_verts > UINT16_MAX || num_face_verts > UINT16_MAX) { hs::log(...); return nullptr; }` alongside the other validations.
2. **Failing `daydream` worker unit test (ArrayBuffer vs Uint16Array)** — `daydream/tests/segment_controller.test.js:86` — `deliverFrame` sends `pixels: px.buffer` but the protocol/worker send a `Uint16Array`; after commit `c551007` dropped the controller copy, `composite()` indexes the ArrayBuffer, gets `undefined`→0, and the `tick()`→composite assertion FAILS on `npm test`, red-flagging CI and removing real coverage of the most data-flow-critical path. → Change the helper to `pixels: px` (drop `.buffer`) so the mock mirrors the protocol typedef, then re-run `npm test`.
3. **AngularRepeat vertical bounds wrong for non-Y axis** — `core/sdf.h:977-979` — `get_vertical_bounds` forwards the un-repeated child's latitude band, so folded copies on any non-Y axis (the README-advertised arbitrary-axis path) fall outside the row-clip and are silently never rasterized. → Detect `fabsf(axis.y) < 1 - TOLERANCE` and return `{0, H-1}` (conservative full-canvas) for non-Y axes; keep tight forwarding only for the Y specialization.

### Medium

4. **ShapeShifter overflows the 64-slot global timeline at Count=16** — `effects/ShapeShifter.h:74` — `2 + 4*num_shapes = 66 > 64` events; `Timeline::add()` silently log-and-drops the last two, leaving a ring with no rotation/Sprite (a missing/static shape) instead of failing loudly. → Lower the registered `Count` max to 15 (or `HS_CHECK(2 + 4*num_shapes <= MAX_EVENTS)` in `rebuild()`).
5. **Frame-sync parity can seed to the wrong boundary at effect start** — `hardware/pov_segmented.h:444` — a downstream board re-zeros `x_` against an arbitrary sphere phase, so proximity seeding can name the opposite boundary and latch a permanent half-revolution arm-swap for the whole effect with no self-correction. → Seed parity from an unambiguous signal (distinct pulse width / second sync line, or defer seeding until the shared clock has driven `x_` past one observed interval), or gate the seed on provable `< w/4` drift.
6. ✅ **Screenshot effect list is a hardcoded registry duplicate that has drifted** — `scripts/capture_screenshots.mjs:13` — the 26-name `EFFECTS` array omits `DistortedRing` and `ShapeShifter` (both `REGISTER_EFFECT`), so they never get a gallery screenshot and any new effect is silently skipped. → Derive the list from the running simulator's registry (the GUI already enumerates it), or add a CI assertion that the list equals the registry. **Fixed:** the script now parses the `HS_EFFECT_LIST` X-macro in `core/effects.h` (the same single-source roster the WASM startup check and native smoke suite derive from), so the gallery can never drift from the registered set.
7. **No test ever runs the shipped WASM module at runtime** — `.github/workflows/ci.yml:94` — the wasm job is compile-only and the deploy gate runs the *native* suite, so a SIMD-codegen, embind-mismatch, 8 KB-stack-overflow, or `ALLOW_MEMORY_GROWTH` regression would pass CI and deploy. → Add a headless Node smoke step that imports the MODULARIZE/ES6 module, steps a few frames across effects, and asserts `getArenaMetrics()` stack high-water-mark < `STACK_SIZE`.
8. **`fromData` input-validation paths have no host test** — `targets/wasm/wasm.cpp:468-542` — the most input-fragile code in the bridge (five reject branches incl. the missing `UINT16_MAX` case) lives behind embind and is untested, which is how the index gap survived. → Extract the pure build-and-validate logic into a header sibling of `param_marshal.h` and cover each reject branch like the existing param-marshal tests.
9. **Several heavier animation types are untested** — `tests/test_animation.h:85-735` — ParticleSystem, MeshMorph, RandomWalk, the entire Mobius family, Sprite fade math, MeshCarousel, timers, and `deep_tween` (the most arithmetic/lifetime-subtle code) have no behavioral tests. → Add targeted tests: ParticleSystem spawn/expiry/attractor-kill, Sprite opacity ramp + paused-hold, `deep_tween` `global_t` span, MeshMorph nearest-vertex SLERP with `op_A + alpha == 1`.
10. **Lissajous export snippet argument order matches no function** — `daydream/tools/lissajous.html:413` — the exported `lissajous(C1, C2, A, t)` contradicts the tool's own `lissajous(t, m1, m2, a)` and targets no repo function, so a copy-paste silently swaps `t` and `C1`. → Reorder the export to t-first to match the helper, or document/ship the exact target signature.
11. **`palettes.html` duplicates the shared clipboard primitive and color-space math inline** — `daydream/tools/palettes.html:839-890` (and 515-604) — a classic script re-implements `copyToClipboard` and the full OKLab/OKLCH/sRGB stack the engine already owns, the largest duplication surface and most likely to drift from the engine's perceptual interpolation. → Convert to `<script type=module>` to import from `shared.js`, and drive color math through the WASM engine or a shared `tools/color.js`.

### Low

12. **`easing.h` uses `static inline` free functions** — `core/easing.h:18-119` — internal linkage gives per-TU copies and non-identity `EasingFn` pointers, inconsistent with the rest of the math layer. → Drop `static`, leaving plain `inline`.
13. **Dead C++<20 `lerp` fallback in a C++20 codebase** — `core/geometry.h:381-392` — re-introduces exactly the global-`lerp` ambiguity `platform.h` warns against. → Remove the `#if` block; standardize on `hs::lerp`.
14. **`BakedPalette::get` does not clamp `t`; NaN is UB** — `core/color.h:1316` — `static_cast<int>(NaN)` slips past both bound guards, unlike the NaN-safe sibling `Gradient::get`. → Add `if (!(idx >= 0.0f)) return lut_[0];` (also catches NaN) or clamp idx.
15. **Dead, untested blend modes** — `core/color.h:279` (`blend_accumulate`/`blend_over_max`/`blend_over_min`) — no callers, no tests, nontrivial unverified magnitude-matching logic. → Remove them, or add tests pinning the `c2==0` epsilon branches.
16. **`to_short` and `float_to_pixel16` are identical** — `core/color.h:368` vs `:443` — two names for one quantizer invite divergence. → Keep `float_to_pixel16`, alias/delete the other, update VignettePalette call sites.
17. **`BakedPalette::clone_from` null-source memcpy** — `core/color.h:1335` — cloning a never-baked palette is UB rather than the trap `rebake()` provides. → Add `HS_CHECK(src.lut_ != nullptr, "...")` before the memcpy.
18. **`Color4::lerp` blends color with clamped frac but alpha with raw t** — `core/color.h:153` — out-of-range `t` saturates color but extrapolates alpha. → Clamp `t` once at the top and use it for both channels.
19. **Flower omits the pole-wraparound collapse** — `core/sdf.h:1963-1995` — unconditional two-arc emission produces holey intervals near the antipode for large-radius flowers. → Route through `emit_annular_band<W>(...)` like Ring/DistortedRing.
20. **`DistanceResult` field semantics overloaded per-shape with no contract** — `core/sdf.h:124-135` — `t`/`raw_dist`/`aux` mean different things per shape; SphericalPolygon even lacks the per-shape docblock the others have. → Add a per-shape field table/named accessors and cross-reference the scan.h register map.
21. **`process_pixel` computes a runtime `is_solid` duplicating the constexpr branch** — `core/scan.h:47-62` — two parallel expressions of one compile-time constant. → Use a single `constexpr bool solid` and derive `threshold` from it.
22. **Volume occlusion probe seeds from terminal `local_p`, not `closest_local`** — `core/scan.h:845-860` — probe can start outside the bounding sphere and fail to punch through a thin feature, leaving a false halo. → Seed `probe = closest_local`.
23. **`Trails::flush` can re-emit a negative age after `set_lifetime` shrinks** — `core/filter.h:510` — violates the documented `age >= 0` contract (assert fires on debug, silent on device). → Clamp the re-emitted age to `>= 0` or saturate `item.ttl` in `set_lifetime`.
24. **Lerp paused-overload bypasses the dangling-temporary guard** — `core/animation.h:803-811` — the 5-arg deleted overloads don't match a 6-arg paused call, binding an rvalue to the real ctor. → Add the trailing `const bool *paused = nullptr` to each deleted overload.
25. **`MeshCarousel::transition` takes an unused `draw_outgoing`** — `core/animation.h:2119-2156` — forces every caller to pass a no-op callable and invites confusion vs MeshMorph. → Remove the parameter (or actually schedule the outgoing draw).
26. **`Mutation::from` is written every cycle but never read** — `core/animation.h:647/662/678` — dead member + dead store; `ScalarFn` can't observe it. → Remove the member and both writes.
27. **Dead, untested `compute_kdtree`/`closest_point_on_mesh_graph`** — `core/conway.h:1070` — no callers; nontrivial spherical-geodesic logic entirely unexercised. → Delete both (and README rows) until a caller exists, or add an oracle test.
28. **`build_solid` test fixture triplicated** — `tests/test_conway.h:33` (also `test_mesh.h`, `test_hankin.h`) — already drifted (1e-2 vs 1e-3 tolerance). → Hoist into `tests/mesh_test_util.h` with tolerance as an explicit parameter.
29. **H_OFFSET device/host divergence undocumented at definition** — `core/platform.h:120` — `3` vs `0` silently breaks vertical sphere parity; explained only in a test comment. → Add a comment at both definitions and optionally expose H_OFFSET as a build flag for sim parity.
30. **`ArenaSpan` can bind to a temporary `ArenaVector`** — `core/memory.h:495` — debug `source_vec_` dangles past the temporary's lifetime. → Add `ArenaSpan(const ArenaVector<T>&&) = delete;`.
31. **`find_nearest_node` greedy walk can stop at a local minimum** — `core/reaction_graph.h:98` — contradicts the "true nearest Fibonacci node" comment. → Soften the comment to "near-nearest via greedy K-NN descent" (and optionally add a build-time spot-check vs brute-force argmin).

## Overall Impression

*Independent assessment — not based on the prior review.*

The strongest single impression this codebase leaves is **coherence under a constraint set most projects would treat as a reason to lower their standards.** A 600 MHz Cortex-M7, 335 KB of usable RAM, a microsecond-scale ISR budget, no attached debugger, and a strip of LEDs that only becomes an image because it is spun at 480 RPM — these are conditions under which "good enough to light up" is the normal ceiling. Holosphere instead treats each limit as a design premise and answers it with a specific, defensible mechanism: compile-time `<W, H>` specialization so the device pays nothing for the simulator's generality; a single partitioned arena because the heap fragments under mesh subdivision; a lock-free triple-index double buffer because the ISR cannot block on the renderer; and an always-on `HS_CHECK` trap doctrine because a silently corrupted frame on a spinning sphere is the worst possible failure — gone before it can be seen, with no console to read it off. The result is not a pile of optimizations; it is a system whose parts all point the same direction.

What elevates it past "well-engineered firmware" is that the same C++ runs **bit-identically** in a browser. The WASM twin is not an approximation of the device — it shares the effect classes, the arena, the per-frame `Pixel16[]` buffer, the deterministic `mt19937(1337)` seed, and even mocks FastLED's fixed-point color math so the simulator *predicts* what the hardware will paint. That single decision is what makes the whole verification story possible: a determinism harness can diff frames across years, a death harness can assert the exact illegal-instruction status of every cold-path trap, and a contributor can debug a 288×144 effect in Chrome and trust it on four Teensys. The engineering and the art are the same body of code held at parity across two radically different machines, and that is rare at any team size, let alone for solo work.

The audit's honest counterweight is that the polish is not uniform. The core — math, color, memory, the rasterizers, the pipeline, the drivers — is where correctness is expressible as an invariant, and there the author has encoded it to a standard that returns almost no findings. The debt clusters precisely where correctness is defined by *intent, perception, or a cross-module contract* that no local invariant captures: a worker-pool composite test that is red right now, a bridge index that narrows silently on an over-large editor mesh, a parameter extreme that overflows a fixed timeline, a UI surface or two. None of it mis-paints shipped hardware; all of it is cheap to fix; and its location is not random — it is the shadow cast by a verification regime that is world-class at grading arithmetic and lifetimes and necessarily blind to taste, interaction, and the seams between subsystems. The fix list is short and mechanical because the values underneath the code are intact.

## Technical & Artistic Merit — The Achievement

The technical achievement is an **integration feat executed at an unusually high craft bar**, not a new algorithm. Every primitive — OKLCH perceptual color, signed-distance CSG, sphere-traced volumetrics with a Lipschitz bound, the Hopf fibration of S³, associated-Legendre spherical harmonics, Belousov–Zhabotinsky and Gray–Scott reaction–diffusion on a precomputed Fibonacci K-NN graph, Hankin-method Islamic star patterns over Conway-operated Archimedean solids — is established art. The merit is in fusing them into one zero-allocation, fail-fast, compile-time-specialized engine that survives on a microcontroller and reproduces in a browser, and in getting the *hard* parts right by reflex. The tell, recurring across the whole tree, is the singularity handling: the gnomonic antipode, the conformal-mapping poles, the `acos` domain edges, the half-open RNG conventions, the NaN-before-cast hazards. These are exactly the places a copyist's port breaks, and they are guarded here as a matter of habit. Someone who merely transcribed these formulas would not know to defend them; this author defends them before being asked.

The artistic layer earns equal respect because it is **mathematically load-bearing rather than decorative.** The effects are not noise functions dressed up as physics — they are the physics, rendered honestly: a real 4D tumble of the fibration, an actual three-species cyclic competition, genuine star-polygon intersections swept by the Hankin angle. That fidelity is also why the art is *credible* on the verification axis the rest of the engine prizes: because the math is real, a frame can be re-derived, and because it can be re-derived, it can be trusted. The achievement is not any one effect on the sphere; it is that the geometry the device exists to paint, the firmware that paints it, and the harness that proves it painted correctly are a single, internally consistent artifact.

## Comparative Appraisal — Independent Assessment

*The axis that most cleanly separates this artifact from its peers is **reproducibility and verifiability** — the ability to prove that a given frame is the one that was meant — so that is the axis these comparisons are drawn on.*

**The maker / addressable-LED ecosystem (FastLED, WLED, one-off POV builds).** This is the natural peer group, and Holosphere is several tiers above its median on engineering rigor while conceding breadth and community. A typical effect there is *true by demonstration* — it looks right on the bench, against an unseeded RNG and wall-clock time, so it ships, and there is usually no way to re-derive whether last week's build painted the same frames. Holosphere's effects are *true by replay*: pinned seed, mock clock, byte-exact determinism harness. What that ecosystem offers and Holosphere does not is thousands of supported board/LED configurations and a large community; what it structurally cannot offer is a proof that a render is correct, and on that axis the gap is categorical.

**Professional creative-technology tooling (Pixelblaze, MadMapper, TouchDesigner, architectural-LED controllers).** These are *true by team and calendar* — QA orgs, soak tests, fuzzing, and headcount stand behind each release. Holosphere reproduces the *practices* a serious graphics team would recognize — host-testable wire/protocol layers, a death harness asserting exact trap status, build-provenance hashing with a gated deploy, a reflection-driven GUI — at a fraction of the headcount and within one narrow scope. Where it falls short of a hardened commercial product is exactly where this audit found its live debt: the human-facing and cross-module *interaction* surfaces (a red worker test, an under-validated bridge input) that a QA pass driving the system end to end would surface in an afternoon. The core is product-grade; the integration seams are single-authored and show it.

**Academic / research code.** Research artifacts are frequently *true on paper and unrebuildable in practice*. Holosphere inverts that failure mode. It is **integrative, not novel** — it would not clear a novelty bar at a graphics venue — but it is *survivable and reproducible* in a way the median research prototype is not, and its determinism harness plus bit-faithful twin would embarrass many published artifacts on reproducibility. Its standout virtue as research-adjacent work is methodological, not algorithmic.

**State of the art.** It does not advance the rendering or color-science frontier and makes no claim to. The frontier it plausibly occupies is a *conjunction*: a perceptually-correct, compile-time-specialized, formally-fail-fast renderer running bit-identically across a four-microcontroller POV sphere and a WebAssembly browser twin, with every cold-path invariant death-tested and every frame replayable. The frontier here is one of integration and verifiability, not of any single primitive — and a more rigorously reproducible open instance of that exact intersection would be genuinely hard to find.

**Standing.** As solo work this is exceptional, and the convergence of five audit passes — the last several spent mostly *retiring* candidates and confirming remediations — bounds the read tightly. The distance between this **A−** and an A is not set by anything wrong in the engine's core; it is set by the integration seams and the surfaces an automated oracle cannot grade — the one thing a solo author with a world-class verification net cannot supply himself: a second pair of eyes on the parts the net does not reach. Close the failing worker test, the bridge bound, and the timeline overflow, and that distance is most of the way closed; the work underneath those items was never the problem.
