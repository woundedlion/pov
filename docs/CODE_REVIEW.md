# Holosphere + Daydream — Code Quality Review

**Date:** 2026-06-07
**Reviewer:** Expert C++/TypeScript code audit — fresh independent pass
**Method:** 16 parallel component agents plus a dedicated README-digest agent, each reading its assigned source in full and citing `file:line`. Findings were judged against the project's own stated philosophy — always-on fail-fast (`HS_CHECK` traps surviving `NDEBUG`, no stdio, `__builtin_trap`), 16-bit linear color with Ottosson-correct OKLab/OKLCH interpolation, compile-time `<W,H>` specialization, and a single partitioned arena with explicit `Arena&` plumbing. This pass **deliberately did not assume any "FIXED" annotation held**; every claim was re-derived from the code as it stands. The lead reviewer then spot-verified the highest-impact new findings (the Liquid2D degenerate presets, the GnomonicStars latent use-after-free, the `truncate(50°)` contract gap, and the dirty provenance working tree) directly against the source and git state.
**Scope:** ~33k lines of hand-written C++ engine (`core/`), 27 modern effects (+ a CRTP reaction-diffusion base), 3 hardware drivers + `platform.h`, the WASM/embind bridge, a 24-module native test suite, the CMake/CI build; plus the sibling vanilla-JS `daydream` web simulator. Generated tables (`reaction_graph.cpp`, `color_luts.h`) and third-party code (`FastNoiseLite.h`) were judged on integration/generation correctness, not line-by-line.
**Out of scope (per review rules):** `effects_legacy.h` and `targets/Holosphere/Holosphere.ino`.

## Overall grade: **A−**

This is an exceptional solo engineering project, and an independent re-derivation confirms the bones rather than undermining them. Across a very large surface the architecture is not merely principled but *consistently applied*: every render class is templated on `<W,H>` for zero-overhead specialization; all internal color is 16-bit linear light with achromatic-guarded, shortest-arc OKLCH interpolation; memory is a single partitioned arena with explicit `(Arena&, Arena&)` plumbing and no hidden state; and the rendering core is a variadic `Pipeline<W,H,Filters...>` that performs automatic 2D↔3D coordinate coercion at compile time with no runtime cost. The performance engineering is expert-level — split sin/cos trig LUTs (~145× RAM win, [geometry.h:209-257](../core/geometry.h#L209)), an anisotropic per-face distance LUT whose validity gate is now the correct 1-Lipschitz per-cell-diagonal bound, Lipschitz-aware sphere tracing of warped SDFs, a branchless segmented ISR, inline 16-bit→wire packing that bypasses the CRGB intermediate, and a zero-copy WASM readback whose detachment contract genuinely holds. The 1,848-line README is a genuinely outstanding engineering document.

Crucially, this skeptical pass **vindicated most of the prior correction campaign** rather than re-opening it. Re-derived from current code: `narrow_index` lives in the shared `mesh.h` `MeshOps` and both `conway.h` and `hankin.h` route every output-index narrowing through it (hankin has zero raw output-index casts); the WASM build carries `-fno-finite-math-only` immediately after `-ffast-math` ([CMakeLists.txt:57](../CMakeLists.txt#L57)) so NaN traps survive; Voronoi is a real per-frame KD-tree with an exactly-equivalent `k=2` query; GS hoists its 7680-node lattice into `init()` like BZ; the OKLab matrices are Ottosson-exact and the 16-bit lerp cancels the `>>16` darkening bias; the deploy gate hard-fails on missing/mismatched provenance and the *committed* `.wasm`/`.sha256` are internally consistent; the live-art pause model is complete; the dual-stamp arena lifetime check (arena generation + per-vector rebind counter) is present and correct; and the segment-compositor stale-resolution OOB is defended on two independent layers. The K-NN reaction table is structurally flawless (7680 rows × uniform K=6, all in range, no self-references).

What holds it at **A− rather than A** is a genuine, coherent correctness cluster — and this independent pass found two real defects a self-review missed:

1. **Liquid2D ships degenerate presets that bite at default parameters.** `Presets<Params,4>` declares four slots but supplies only two entries of six values each, while `Params` has seven fields ([Liquid2D.h:192-195](../effects/Liquid2D.h#L192)). `cycle_speed` is therefore zero in both populated presets, and presets[2]/[3] are *entirely zero*. The init-time random timer cycles all four, lerping the shader into states where `pattern_freq=0`, `time_speed=0`, `warp_scale=0`, and `pole_fade=0` — the last feeds `1/(1+r²/0)` → NaN. Visible as periodic frozen/black/NaN frames in normal use.

2. **GnomonicStars caches a `pin=false` animation pointer into the permanent param list** ([GnomonicStars.h:29-31](../effects/GnomonicStars.h#L29)). `transformer.spawn(...)` routes through `add_get(pin=false)`, so the timeline event's `handled` flag stays false — disabling the very `HS_CHECK(!handled)` compaction trap meant to catch relocation of a retained handle. It survives today only because the `MobiusWarpEvolving` is infinite (`duration=-1`), so no slot is ever freed and compaction never runs. A single future finite animation inserted earlier dangles a pointer the GUI reads every frame.

3. **`truncate` does not enforce its `[0..0.5]` contract, and two live registry entries feed it `50°≈0.873 rad`** ([conway.h:407](../core/conway.h#L407), [solids.h:447,508](../core/solids.h#L447)). At `t>0.5` the two edge-cut points cross past each other, producing self-intersecting cut quads. The `_truncate50d_` names were corrected to *document* that 50° is passed as the fraction `t`, but the rename does not resolve the geometry, and `truncate` still silently accepts the out-of-contract value rather than trapping per the fail-fast doctrine. Whether the crossed faces are intentional tuned art or a leftover degrees-vs-fraction units error is impossible to determine from the code — which is itself the problem.

4. **An active deploy hazard in the working tree.** daydream's `holosphere_wasm.sha` is dirty (`c688690`) while the committed `.wasm`/`.wasm.sha256` are from an older build, and current engine HEAD is `d55fe0b`. The gate binds `.wasm`↔`.wasm.sha256` but **not** `.wasm`↔`.sha`, so committing the lone `.sha` would deploy a binary built from a different source than the gate verifies.

5. A scatter of **latent math-core traps** the fail-fast doctrine should surface rather than hide: `Spherical(origin)` ([3dmath.h:266-271](../core/3dmath.h#L266)), the integer `pixel_to_vector` reading LUTs with no bounds check ([geometry.h:250-257](../core/geometry.h#L250)), `Orientation::upsample(1)` dividing by `count-1` ([geometry.h:499](../core/geometry.h#L499)), and `Screen/World::Trails` with no `lifetime>0` guard ([filter.h:431](../core/filter.h#L431)).

None of these is architectural; all are fixable and most are localized. The core — color, pipeline, memory, mesh kernel, WASM bridge, single-Teensy hardware path — is genuinely shippable. The gap between the stated philosophy and its enforcement is narrower than the previous skeptical pass implied, but the project is still one localized P1 fix, an enforced `truncate` contract, and a tightened provenance gate away from an unqualified A.

## Cross-cutting themes

### 1. Fail-fast is the headline philosophy, and it is now *mostly* enforced

The lookup/registration/config/memory/mesh layers are genuinely fail-fast and verified by re-derivation: `Arena::allocate`/`ArenaVector` trap on OOM/overflow with no-wrap arithmetic, `bind()` guards the multiply, `Solids::get_by_name` traps on miss, `pair_half_edges` traps a non-manifold edge fan, `SpatialHash::query` traps on bucket overflow at parity with `insert`, and the shared `MeshOps::narrow_index` traps every operator's output-index narrowing in *both* `conway.h` and `hankin.h`. The cold/hot split on palette guards is principled — cold seams (`StaticPalette::bind`, the `Gradient` ctor stop bound) trap always-on, per-pixel `get()` paths use stripped `assert` backed by a cold trap.

The residual gaps are narrower than before but real:

| Gap | Locations | Problem |
|---|---|---|
| **`truncate` accepts `t>0.5` without trapping** | [conway.h:407](../core/conway.h#L407); callers [solids.h:447,508](../core/solids.h#L447) | The docstring contracts `[0..0.5]`; two live registry entries pass `≈0.873`, producing crossed/degenerate cut faces. The fail-fast doctrine says this should trap; instead it silently emits self-intersecting geometry that is indistinguishable from intentional art. |
| **Math-core strict-normalize / OOB traps reachable on legitimate-looking input** | `Spherical(origin)` [3dmath.h:268](../core/3dmath.h#L268); integer `pixel_to_vector` LUT reads [geometry.h:250](../core/geometry.h#L250); `upsample(count-1)` [geometry.h:499](../core/geometry.h#L499) | `pixel_to_vector` reads `sin_phi[y]`/`cos_theta[x]` with *no* `HS_CHECK`, unlike the sibling `y_to_phi` which guards — an unclamped pixel coordinate is a silent OOB read, the exact class the doctrine says must trap. |
| **`Screen`/`World::Trails` divide by `lifetime` with no `lifetime>0` guard** | [filter.h:431,574](../core/filter.h#L431) | A `lifetime==0` trail config is a cold authoring error that produces inf/NaN flowing into `blend_alpha`; a one-line ctor `HS_CHECK` would convert it to a bench trap. |

### 2. The live-art interface still accretes inert and half-live controls

`registerParam` binds a live pointer; a slider is "live" iff `draw_frame` actually *reads* that member each frame, and animation-driven members must be `markAnimated` so the GUI renders them as auto-pausing read-outs. The standardized pause model (`Mutation`/`Driver`/`Lerp`/`Sprite` pause gate + `anims_paused_` + `markAnimated`/`markReadonly`) is well-designed and most strong effects use it correctly. But this pass found it applied unevenly:

| Effect | Issue | Cite |
|---|---|---|
| **Whole RD/Hopf/Islamic/SH group** | **Zero** `markAnimated`/`markReadonly` calls despite animation-coupled params, inconsistent with the 9 effects that follow the convention | [HopfFibration.h:23](../effects/HopfFibration.h#L23), group-wide |
| **HopfFibration** | The "Pause Animation" toggle is **inert** — its three `Driver`s and the `Rotation` are constructed with a `nullptr` pause gate, so flow/tumble/rotation keep advancing while paused | [HopfFibration.h:45-54](../effects/HopfFibration.h#L45) |
| **DreamBalls** | `Warp` is **semi-dead** — value-copied into the mobius generator only at sprite spawn (period 288 ≈ 5 s), so dragging it has no visible effect until the next spawn, unlike the four live siblings | [DreamBalls.h:168](../effects/DreamBalls.h#L168) |

Confirmed **live** (the prior dead-slider list re-verified and cleared): MobiusGrid `Rings`/`Lines`, ShapeShifter `Twist`, SplineFlow `Speed`, ChaoticStrings `Cycle Dur`, Moire `Amp`, HankinSolids `Angle`, IslamicStars `Duration`, MindSplatter `Particles` (correctly `markReadonly` telemetry). The build-time-styled `lint_dead_sliders` is a genuine runtime persistence gate (writes an off-value, renders, asserts survival) but cannot detect the semi-dead/latency case (DreamBalls `Warp`) or the missing-`markAnimated` metadata, because the value *does* eventually take effect.

### 3. Latent correctness at the hard seams

The remaining real bugs live where effect lifecycle, mesh topology, and the math core bite:

- **Effects:** Liquid2D's degenerate presets (theme intro #1) is the one defect that bites at default parameters. GnomonicStars' pointer-into-timeline UAF (#2) is latent but the compaction trap that should catch it is disabled (`pin=false`).
- **Geometry registry:** `truncate(50°)` crossed cut faces, unenforced contract (theme 1).
- **Math core:** `Spherical(origin)` trap, `pixel_to_vector` unguarded LUT read, `upsample(1)` div-by-zero. Quaternion `slerp`/`angle_between` use libm `acosf`/`sinf` while the `Vector` paths use `fast_*` — a latent perf inconsistency in `Orientation::upsample`'s loop.
- **Plot:** `Fragment::lerp` interpolates the integer-semantic `v2`/`v3` index registers to fractional values mid-segment ([plot.h:321](../core/plot.h#L321)); any shader keying off `v2` as a discrete selector sees `1.0, 1.4, 1.7…`. Benign for current shaders (the only `v2` consumers read it constant-or-not-at-all) but a latent trap for new effects. The planar arc-length inversion is structurally correct but under-resolved at `LEN_SAMPLES=4`.
- **Hardware:** the POV deadline check `millis() - start < duration*1000` mixes a signed `long start` with unsigned `millis()` and overflows on multi-week uptime ([pov_segmented.h:243,275](../hardware/pov_segmented.h#L243)); the ID-decode `~(...) & (N-1)` is correct only by the `& (N-1)` mask saving a full-width `~` ([pov_segmented.h:178](../hardware/pov_segmented.h#L178)).
- **Web:** pause state is not re-synced to workers/engine on effect switch — pausing effect A then switching to B can leave B frozen while the toggle reads false ([daydream.js:152-211](../../daydream/daydream.js#L152)).

None of the latent items bite at current parameters — which is exactly why they need tests, not just fixes.

### 4. Build provenance: correct as committed, but a live working-tree hazard

The deploy gate is genuinely well-built: `-fno-finite-math-only` is correctly ordered after `-ffast-math` ([CMakeLists.txt:57](../CMakeLists.txt#L57)) so NaN traps survive the WASM build; the committed `holosphere_wasm.sha`/`.wasm.sha256` are tracked and internally consistent; `.gitattributes` forces LF on the provenance files so `sha256sum -c` works on the Linux runner; and the gate hard-fails (no engine-HEAD fallback) on missing/mismatched provenance. **But** the gate binds `.wasm`↔`.wasm.sha256` only — nothing binds `.wasm`↔`.sha` (the engine-source pin). The daydream working tree right now has `holosphere_wasm.sha` dirty (rebuilt to `c688690`) while the `.wasm` is clean from an older build and engine HEAD is `d55fe0b`; committing the lone `.sha` would check out engine source X, pass the sha256 check on a binary built from source Y, and deploy the decoupled pair. This is the one place the otherwise-mature supply-chain story has a gap.

## Standout engineering

Excellence should be named, not just defects:

- **The variadic filter pipeline** ([filter.h:162-191](../core/filter.h#L162)). `Pipeline<W,H,Head,Tail...>` derives from `Head` (EBO, zero storage) and dispatches `plot()` via `if constexpr (Head::is_2d)`, transparently lifting 2D↔3D coordinates at the mismatch boundaries with no runtime cost — the cleanest expression of the engine's compile-time-resolution thesis, funneled through one wrap+clip+blend sink.
- **The anisotropic distance LUT with a per-face validity gate** ([sdf.h:1188,1479](../core/sdf.h#L1188)). Bilinear interpolation is trusted only when all four corners share sign *and* the smallest magnitude exceeds `sqrt(step_x²+step_y²)` — the face's own cell diagonal, which is the exact 1-Lipschitz safe bound for the plane SDF. Large faces correctly fall back to exact evaluation; small faces gain LUT hits.
- **The dual-stamp arena lifetime model** ([memory.h:199,461](../core/memory.h#L199)). The per-vector `rebind_generation_` catches precisely the case the arena generation cannot: a `bind()`-driven re-grow that swaps `data_` without resetting the arena. `ArenaSpan::check_alive` checks both stamps — a debug-only, device-free use-after-free net.
- **The honest relaxed-atomic + cache-flush DMA handoff** ([dma_led.h:279-314](../hardware/dma_led.h#L279)). It separates the single-observer ISR/thread flag (no barrier needed on one core) from DMA buffer coherence (handled by `arm_dcache_flush_delete` + DSB before `dma_.enable()`) and explicitly warns against a bogus acquire/release "fix". Staff-level reasoning that is *correct* — do not "fix" it.
- **OKLab/OKLCH interpolation with achromatic guards and a bias-free 16-bit lerp** ([color.h:431-504](../core/color.h#L431), [color.h:95-123](../core/color.h#L95)). Ottosson-exact matrices, near-zero-chroma special-casing on *both* endpoints before the ±π hue wrap, and a `(x + (x>>16) + 1) >> 16` reconstruction that avoids the `>>16` darkening bias on every blend.
- **The triple-enforced X-macro SSOTs** ([wasm.cpp:115](../targets/wasm/wasm.cpp#L115), [effects.h:50](../core/effects.h#L50)). `setResolution`/`setEffect`/`getEffectSizes` all expand from `HS_WASM_RESOLUTIONS`, and the effect roster is enforced three ways — a startup `HS_CHECK(registry.size()==HS_EFFECT_COUNT)`, the native smoke suite, and the Phantasm firmware show-list — so the supported set provably cannot drift.
- **`SolidBuilder`** ([solids.h:198-275](../core/solids.h#L198)). Each Conway op runs `op(mesh,*a,*b)` then `std::swap(a,b)`, hiding a two-arena ping-pong behind a fluent chain so Islamic-pattern recipes read as a declarative geometry DSL while staying single-allocation and arena-deterministic.
- **The Voronoi KD-tree conversion** ([Voronoi.h:75-94](../effects/Voronoi.h#L75)). On the unit sphere nearest-by-Euclidean equals nearest-by-max-dot, so the per-frame KD-tree `k=2` query returns the *identical* result as the former O(N) per-pixel scan — exact, not approximate — turning `O(W·H·N)` into `O(N log N)` build + `O(log N)` per pixel.
- **The two-layer daydream compositor fence** ([segment_controller.js:114,244](../../daydream/segment_controller.js#L114)). A monotonic `renderGen`/`inflightGen` generation fence drops stale-resolution worker frames (settling the promise so the pipeline never stalls) *and* a per-result bounds clamp backstops it, so a mid-flight resolution change cannot OOB-write the resized buffer.

### P1 — Bites at default parameters, latent crashes, and active hazards

1. ~~**Fix Liquid2D's degenerate presets**~~ — **INVALID (validated 2026-06-07).** Re-derived against C++ aggregate-init semantics and confirmed empirically by compiling the exact `Presets<Params,4>` initializer ([Liquid2D.h:192-195](../effects/Liquid2D.h#L192)): because `Params` carries default member initializers, the short initializer lists do **not** zero the missing fields — `cycle_speed` is `0.05` in every preset, and the two trailing slots receive the full default `Params`, not all-zero. There is no `pattern_freq=0`/`time_speed=0`/`pole_fade=0` state and no NaN. The only residual is cosmetic: slots 2–3 duplicate the default (== slot 0), so the table cycles through 2 distinct presets rather than 4. Not a defect; no fix applied.

   - **`Trails` `lifetime>0` — FIXED.** `lifetime` is a per-frame fade divisor; a zero/negative trail length feeds inf/NaN into the blend. Added `HS_CHECK(lifetime > 0)` to both ctors ([filter.h:393,550](../core/filter.h#L393)) — a free cold (construction-time) guard. All current callers pass positive values (Dynamo range `[1,100]`, SplineFlow `60`).
   - **`Spherical(const Vector&)` — keep the trap (recommendation rejected).** The origin is a contract violation, not a legitimate geometric edge (a sphere direction is never the zero vector), so the existing `normalize()` `HS_CHECK` is the *correct* fail-fast behavior. Adding a defined-fallback azimuth would **mask** an invariant violation, contradicting the project's crash-on-invariant doctrine. No change.
   - **Integer `pixel_to_vector` — no per-pixel guard (perf).** This is a per-pixel hot path (`Scan::Shader`, Voronoi, Metaballs). A per-pixel `HS_CHECK` would regress the hot loop for zero added safety. No change.

A guard before the early-return would wrongly trap valid `count==1` calls; after it would be dead code. No change.

## Comparative appraisal

*This appraisal is derived independently from the current code, not from any prior assessment.*

**Versus hobbyist / maker LED-art projects (the immediate peer group):** Not comparable — several leagues above. The typical FastLED POV project is a single `.ino` of gamma-corrected 8-bit blends, `delay()`-driven timing, and copy-pasted effect loops. Holosphere has a typed compile-time rendering architecture, 16-bit linear light with perceptual OKLab/OKLCH color science, an arena memory model with always-on fail-fast invariants and a dual-stamp use-after-free net, a half-edge mesh kernel implementing the full Conway-operator algebra and Hankin's-method Islamic patterns, a WASM digital twin compiled from the *same* engine source, a multi-Teensy segmented ISR with documented cache-coherency reasoning, and an invariant-driven native test suite that proves its safety traps actually fire. This is top-0.1% for the category and not close.

**Versus professional / commercial offerings:** It compares favorably with the *engine* quality of commissioned generative-LED installation software, and on color science and memory discipline exceeds much of it. Where a shipping product still pulls ahead is the operational envelope: CI on real hardware (the device's 335 KB ceiling and 32-bit layout are structurally invisible to the 8 MB host test build), a QA pass that would have caught Liquid2D's degenerate presets and DreamBalls' laggy slider before they shipped, and the provenance discipline to never let a deployed binary decouple from its tested source. The *core* is genuinely shippable; the *seams* — one effect's preset table, a latent UAF, an unenforced geometry contract, a working-tree provenance skew — are where it still reads as one expert's project rather than a team's hardened product.

**Versus academic work:** The numerical maturity is at or above a strong graphics/geometry-processing course project or a SIGGRAPH-adjacent demo: matched projection pole sentinels behind named SSOT constants, analytic Lipschitz bounds for safe sphere-tracing of warped SDFs, Ottosson-correct OKLab/OKLCH shortest-arc interpolation, Conway-operator + Hankin-pattern computational geometry on a half-edge kernel, a provably-correct Fibonacci-lattice K-NN reaction graph, and a stable spherical-harmonic recurrence. What it lacks is the academic apparatus — no novelty claim, no baseline evaluation, no ablation. It is *engineering* of known-good algorithms executed unusually well, not a research contribution. As the artifact accompanying a thesis on real-time spherical rendering it would be a standout.

**Versus the state of the art:** The individual techniques are well-chosen current practice rather than frontier (split LUTs, instanced rendering, embind zero-copy readback, EBO variadic templates, sphere tracing, KD-tree nearest-neighbor). The *synthesis* is what impresses: one unchanged C++ engine driving both a microsecond-budget multi-Teensy ISR firmware and a browser digital twin from a single source, with the simulator and the firmware enforcing the same effect roster from the same X-macro. That is genuinely sophisticated systems design and is where the project is most distinctive relative to anything off-the-shelf.

### Technical and artistic merit

**Technically**, this is the work of an expert fluent across the whole stack — Cortex-M7 cache coherency and eDMA, template metaprogramming and EBO, color appearance models, computational geometry, the WASM/JS FFI, and arena memory design — who maintains a *consistent design philosophy* across nearly all of it. The recurring weakness is not ignorance but uneven enforcement at the edges: the fail-fast rule that governs memory/lookup/mesh is not yet extended to `truncate`'s contract or a handful of math-core entry points; the live-art pause model that hardens most effects is unwired in Hopf and unflagged across an entire effect group; and one effect shipped a malformed preset table that the otherwise-strong test harness can't catch because it isn't a trap or an invariant. Close those edges — enforce the contracts, finish the pause-gate wiring, fix the two effect defects, and tie the provenance `.sha` to the binary — and the correctness grade moves from B+ toward A−, and the project with it.

**Artistically**, the effect catalog is ambitious and coherent: reaction-diffusion (Gray-Scott and Belousov-Zhabotinsky) on a Fibonacci-lattice graph, the Hopf fibration with a 4D tumble, procedurally-generated authentic Islamic star patterns via Hankin's method, volumetric raymarched metallic forms, Möbius-warped wireframes, real spherical-harmonic lobes with zero-crossing crossfade. These are not stock patterns, and the engine primitives are expressive enough that the strongest effects (`Raymarch`, `MeshFeedback`, `HopfFibration`, the Liquid2D/Flyby shaders) achieve real visual sophistication in ~25–80 lines apiece. The medium itself — a sphere painting full-color 16-bit-linear imagery in persistence of vision — is novel, and the engineering is in genuine service of the art.

**Bottom line:** **A−.** A remarkable, near-professional codebase. The architecture, performance engineering, memory model, testing, and documentation are all genuinely A−/A grade, the core is shippable, and an independent, skeptical re-derivation confirmed that the prior correction campaign largely held — `narrow_index` routing, the fast-math ordering, the Voronoi/GS rewrites, the OKLab math, the committed provenance, the complete pause model, and the dual-stamp arena net are all real. It is held below A by a correctness cluster this pass surfaced from current code: a Liquid2D preset defect that ships degenerate frames at default parameters, a latent GnomonicStars use-after-free with its safety trap disabled, an unenforced `truncate` contract feeding two degenerate live solids, a scatter of latent math-core traps, and an active provenance working-tree hazard. None of it is architectural — it is one focused round of edge-enforcement and two effect fixes away from an unqualified A.
