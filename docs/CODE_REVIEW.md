# Holosphere — Comprehensive Code Quality Review

**Date:** 2026-06-21 (eleventh audit; supersedes the tenth audit of 2026-06-13, whose findings 433–600 were reconciled and are nearly all marked resolved in the tree — see §5).
**Tree:** Holosphere `d867182`; daydream reviewed at its current `master`.
**Scope:** The Holosphere C++ engine and firmware — `core/` (rendering engine), `effects/` (27 effects + the shared `ReactionDiffusionBase.h`), `hardware/` (POV/DMA/1-wire-sync drivers), `targets/` (WASM bridge + Phantasm entry), `tests/` (native Clang suite), build/CI tooling (`CMakeLists.txt`, presets, `scripts/`) — **and the daydream web simulator** (`daydream.js`, `driver.js`, `state.js`, `recorder.js`, the segmented-POV worker pool, and the standalone geometry tools). Out of scope by instruction: `core/effects_legacy.h`, `targets/Holosphere/Holosphere.ino`, `core/rotate.h`. `core/FastNoiseLite.h` and `core/inplace_function.h` (vendored), and `core/color_luts.h` + `core/reaction_graph.cpp` (generated data) were assessed for integration/provenance only, not line-by-line.
**Method:** Fifteen independent component reviewers, each reading every in-scope file in full with the README as shared architectural context, grading objective quality (correctness, UB, ISR/memory-ordering, bounds, overflow, performance, testability) and subjective quality (architectural elegance, interface expressiveness, idiom). The prior `CODE_REVIEW.md` was deliberately **not** consulted during examination; it was reconciled only afterward (§5). The overall appraisal in §6 is written independently of the prior review.

## 1. Executive Summary

Read cold against the current tree by fifteen reviewers spanning both repositories, the Holosphere/daydream product holds the high bar its audit series has established. **No critical defect, and no correctness bug that ships a wrong frame on the default single-instance render path, was found.** The single most severe item this pass is *latent*: a pool-slot leak in the geometry-transformer layer that only manifests if an effect repeatedly spawns a *perpetual* (infinite-duration) warp animation, because the slot is freed through a completion callback that such an animation never fires.

The signature of this codebase — visible independently in nearly every reviewer's notes — is that its residual defects are overwhelmingly **internal inconsistencies with its own exacting standards** rather than ordinary bugs. The engine's doctrine (fail-fast `HS_CHECK` on cold seams, stripped `assert` on hot paths backed by a cold trap, byte-documented arena budgets, wrapped trig phases, baked-palette LUTs, single-source-of-truth rosters, deleted rvalue overloads enforcing borrow contracts) is applied so uniformly that the way you find a flaw is to locate the one place a rule *isn't* honored: the one free-function divisor without the `> 0` guard its templated sibling enforces, the one transformer whose slot-free path can't fire, the one shader pair kept "in sync by hand" with undocumented divergence, the one per-frame re-bind whose safety rests on an assert that `NDEBUG` strips on the device. When a codebase's defects are best described as exceptions to its own rules, the rules have largely won.

**The grade is A−.** The distance to a flat A lives almost entirely at two frontiers:

1. **Test coverage is the single grade-limiting dimension.** The suite is genuinely excellent where it reaches — independent analytic oracles, a forked death harness that proves every `HS_CHECK` trap fires with the exact illegal-instruction signal, a fault-injecting four-board sync simulator, byte-exact cross-run effect determinism under an injected clock. But the most numerically delicate paths are unpinned: the entire `Scan::Volume`/`TransformedVolume` ray-march used by Raymarch, `Plot::SplineChain`/`ParticleSystem`, the `Pixel::Feedback` non-identity warp+upsample, the reaction-diffusion stability envelope, and 21 of 27 effects (smoke + determinism only, no per-effect output invariant). `configure_arenas()`'s *successful* repartition — the mechanism the README markets for effects to claim memory — is exercised only on its trap path.

2. **A handful of latent reachable hazards and per-frame cost cliffs.** The transformer slot leak; an SDF union interval accumulator sized for one child where a nested union of two needs double; an unenforced "sole-ownership" precondition where a co-driven `Orientation` is clobbered each cycle; `Voronoi`'s per-pixel 400-site KD-tree query with no spatial-coherence reuse; `ShapeShifter`/`GnomonicStars`/`Raymarch` per-frame raster costs that scale with a slider rather than the pixel budget; and, in daydream, stale axis labels that linger when the live label count drops to zero and a worker that ignores `setResolution`'s failure return.

**Verdict: A− overall.** The math/color/memory core, the hardware protocol layer, the WASM bridge, and the test *infrastructure* each grade A or above on their own terms; the Phantasm 1-wire/flywheel sync subsystem and the build/CI engineering are A+. The daydream client matches the engine's discipline — the WASM detach-after-heap-growth hazard, the worker generation fence, and the fault-latch deadlock-breaker are all handled correctly and defensively.

## 3. Component Assessments

**core: math / geometry / spatial** — High-quality, professionally-engineered numerical code, and the strongest component on documentation (A+). Pole/antipode/singularity handling is systematic and consistent: slerp antipodal/near-identical branches, stereo/gnomonic pole sentinels with matched recognize thresholds, NaN-safe `fast_atan2` origin guard, the `normalized()` trap vs `normalized_or()` soft-degrade split applied per call site with documented justification. The split TrigLUT (per-column θ + per-row φ, ~145× memory reduction) is an elegant MCU fit. `Vector` is deliberately trivially copyable for `memcpy` fast paths, with the dependency stated and `=default`-enforced. No Critical/High defects; residuals are micro-nits (`std::abs` of a sum of squares, an inline conjugate, a discarded KD-tree distance).

**core: sdf / scan** — The scanline-culling architecture is genuinely good: analytic per-row interval generation avoids `distance()` on empty columns, shared emitters keep the byte-identical math in one place, and the seam-coalescing logic (negative-start re-wrap, the `static_assert`-proven 2× split bound) is carefully reasoned. The premultiplied-alpha SSAA accumulation and the per-pixel Fragment reset to kill ghosting are subtle and right. The one real bug is a sizing mismatch (union accumulator capped for one child, traps on a nested two-child union); the rest are cosmetic AA/precision nits. Test coverage is the weak point — the volumetric ray-march path has no direct test.

**core: plot / waves / easing** — Adaptive stepping is well-engineered (`sin(φ)` density scaling with a pole floor, the cumulative-arc-table inversion that restores arc-uniformity under the azimuthal-equidistant metric — a subtlety most implementations get wrong), and the clip-cull bulge handling computes closed-form great-circle latitude extrema rather than testing endpoints. The substantive finding is documentation: Star/Flower/PlanarPolygon expose a *geodesic* arc-length register while rendering *planar*, a mismatch documented only for Multiline. Everything else is latent-trap or missing-symmetry-guard tier.

**core: color** — The linear/sRGB and OKLab/OKLCH stack is correct and thoughtfully factored: 16-bit linear blending, shortest-arc hue interpolation with achromatic-endpoint handling, a chroma-preserving gamut clip gated behind an in-gamut fast path, no `powf`/`atan2` on any hot path. `lerp16`'s shift-reconstruction is exact at both endpoints and overflow-free (verified exhaustively). Premultiplication discipline is explicit and deferred to the single canvas write; `explicit operator CRGB()` turns an accidental gamma round-trip into a compile error. Residuals are cold-path divide-by-zero gaps in a few palette modifiers.

**core: animation / transformers / generators** — Exceptional defensive commentary and compile-time-enforced borrow contracts (deleted rvalue overloads on `Lerp`/`Motion`/`MobiusFlow`/`MeshMorph`), correct `std::launder`/`iface`-upcast type erasure for `TimelineEvent`, and an O(active) orientation-collapse pre-pass. This is also where the most severe finding lives: the `Transformer` pool frees a slot only through a completion callback, which a *perpetual* spawned animation (`MobiusWarpEvolving`, default infinite duration) never fires — a latent slot leak. Plus an unenforced Motion sole-ownership precondition and an unclamped `Path::get_point`.

**core: filter / canvas / concepts** — The variadic `Pipeline<W,H,Filters...>` with compile-time World↔Screen↔Pixel domain dispatch is the strongest single design in the engine: three `static_assert`s encode the non-obvious ordering invariants directly into the type system. AntiAlias weight handling (floor-based seam split, quintic easing, `sin(φ)` density compensation, surviving-row renormalization) is careful and correct, and the Feedback delta-encoding/seam re-centering is a subtle, correct fix. The 2-physical-buffer + `buffer_free()` gate is correct by design (confirmed, not a tearing bug). Minor: a conditionally-live `col_used[W]` array that is one edit from an uninitialized read.

**core: mesh / conway / hankin / solids** — A mature, self-consistent subsystem. Every topology index cast routes through a trapping `narrow_index`/`narrow_face_count`; every hand-rolled half-edge walk carries an anti-hang `HS_CHECK`; capacity sizing is closed-form and exact per operator; the `(Arena& target, Arena& temp)` polarity contracts are documented where they are load-bearing. Residual defects are all Low and all unreachable from the trusted solids registry — they open only if untrusted/malformed mesh input is introduced (the WASM editor path), which the code's own comments already flag.

**core: memory / platform / registry** — Best-in-class on clarity (A+) and the safety foundation of the whole engine. Arena overflow math is defense-in-depth (subtractive no-wrap bounds plus an independent multiply-overflow trap), alignment is validated before use, the single partitioned block is maximally aligned, ScratchScope LIFO is structurally enforced, and the static-init-order fiasco is correctly avoided in the registry (Meyers singleton + `used`/`retain` against `--gc-sections`). The `platform.h` FastLED mocks are bit-faithful, not approximations. The `beat88` 64-vs-32-bit wrap is the one real divergence from the otherwise-airtight sim==device contract.

**effects** — 27 effects that lean on engine primitives (KDTree, ParticleSystem, Plot/Scan, baked palettes, `MeshCarousel`, `deep_tween`, OrientationTrail) rather than reinventing them; the CRTP `ReactionDiffusionBase` is exemplary factoring. Numerical hardening is thorough (pole/singularity guards, phase wrapping for multi-day runs). The weak dimension is performance budgeting: a few effects (Voronoi's per-pixel KD-tree, ShapeShifter's ~256 rasterizations, GnomonicStars' 2000 unculled stars, Raymarch's `O(verts·steps·W·H)`) carry per-frame cost that scales with a slider rather than the pixel budget, some acknowledged in-code and some not.

**hardware** — The crown jewel. The flywheel-from-time architecture (deriving column position from the CPU cycle counter rather than counting ISR fires) makes masked-IRQ windows lossless *by construction*, and the per-half-rev epoch fold makes the 32-bit counter wrap structurally unobservable. The single-writer ISR model is real and rigorously enforced; the count-coded odd-only symbol alphabet moves single-edge errors to "missed, never wrong"; memory-ordering rationale is unusually rigorous and correct for single-core M7; pure host-testable cores are cleanly split from device shells. The one suspected edge is an epoch×beacon `publish_build` interaction during the construction window, worth a dedicated test.

**targets: wasm / build / scripts** — Single-source-of-truth discipline is exemplary and pervasive (param order, resolution list, effect roster all derive from one expansion, so drift is engineered out). The zero-copy memory-view contract is understood end-to-end (pre-sized once, never realloc'd, re-fetched on detach), the untrusted-JS boundary consistently rejects-and-returns rather than trapping, and the LUT generator matches the C++ reference bit-for-bit. The `-ffast-math`/`-fno-finite-math-only` interaction is correctly diagnosed. Gaps are in CI breadth (the embind param layer and a LUT-regeneration diff are uncovered) and two stray untracked files at repo root.

**daydream: web core** — Matches the engine's discipline. The hardest hazard in a zero-copy WASM bridge — detach-after-heap-growth — is handled rigorously and in the right places, the three-way alias invariant is asserted rather than assumed, the URL-sync race is a single-owner read-modify-write, and listener/observer/worker/GPU teardown is genuinely symmetric. One real visible bug: stale axis/effect labels linger when the live label count drops to zero (CSS2DRenderer only hides during a render pass that the code skips at zero labels).

**daydream: workers + tools** — The generation fence (snapshot-at-dispatch, compare-at-receipt) is a clean, correct solution to stale-frame compositing, and the fault latch that breaks the would-be deadlock when a WASM trap fires `onerror` but never `frame` is genuinely well-reasoned and fully tested. Zero-copy transfer discipline is correct (fresh per-frame buffers, no cross-frame aliasing). The C++-codegen string builders are locked byte-for-byte by tests — important since they paste into the engine. Main gap: the worker ignores `setResolution`'s failure return, and `segment_worker.js` itself has no unit test.

## 4. Prioritized Fix List

Every defect found this pass, grouped by severity, numbered sequentially. Each entry: `file:line — title. problem → fix`.

### Critical
*(none found)*

### High

1. ✅ `core/transformers.h:154` + `core/animation.h:2007` — **Perpetual spawned animations leak transformer pool slots.** `spawn_impl` frees a slot only inside the `p->then(...)` completion callback, but `MobiusWarpEvolving` (the type behind `MobiusWarpGnomonicTransformer`) is default-constructed with infinite duration, so `done()` never becomes true, `then()` never fires, and the slot is never released. After `CAPACITY` spawns the pool is exhausted and further spawns silently return `nullptr`. Latent — requires an effect to repeatedly spawn a perpetual transformer; not confirmed reachable on a shipped effect. → Give the spawned animation a finite/self-resetting duration, or have `Transformer` detect a perpetual animation and route it through an explicit `release(idx)` instead of `then()`; at minimum `HS_CHECK` that the spawned `AnimT` is finite.

### Medium

2. ✅ `core/sdf.h:740,847` — **Union/SmoothUnion per-row interval accumulator is half the size a two-child union can need.** `merged` is capped at `kIntervalSpanCap` (32), but it collects from *both* children, each of which can itself emit up to 32 intervals; a nested union near the cap pushes up to 64 and `push_interval`'s `HS_CHECK` traps on valid geometry. Latent (shipped effects don't build such deep CSG). → Size union accumulators to `2 * kIntervalSpanCap` and replace the literal `32` with the named constant (also at 950–951, 1127–1128).
3. ✅ `core/animation.h:1300` — **Motion drift-anchor reset clobbers a co-driven Orientation.** The "sole-ownership invariant" the reset relied on was unenforced; if two animations co-drove one `Orientation`, `orientation.set(anchor_)` overwrote the co-driver's accumulated rotation each cycle. → Resolved by removing the invariant rather than enforcing it: Motion now integrates by *relative* deltas between consecutive path frames (a pure function of the path parameter — point + tangent, pinned to a full orthonormal frame via the new `quaternion_from_basis`), so its per-cycle contribution telescopes drift-free and it never writes the Orientation's absolute state. The cycle re-seat (for open paths) is now a relative delta carried across the seam, so a co-driver's rotation rides along instead of being clobbered. The anchor/absolute-reset and the sole-ownership precondition are gone; covered by `test_motion_codriven_survives_repeat_seam`.
4. ✅ `core/animation.h:79` — **`Path::get_point` does not clamp `t`.** Out-of-`[0,1]` input is memory-safe (guards catch it) but silently returns the wrong endpoint instead of clamping or trapping, unlike every other interpolator in the file. → `t = hs::clamp(t, 0, 1)` at entry.
5. ✅ `effects/Liquid2D.h:16` / `effects/Flyby.h:18` — **Hand-synced stereographic shader pair with undocumented divergence.** The two `sample()` bodies must stay in sync "by hand," and the warp time-scale factors that differ (0.3 vs 0.5) are uncommented while the header note overstates the rest of the sync contract. → Extract the shared body into a `StereoShaderBase`/free function, or at minimum document the divergent constants at their call sites. *Resolved as doc-only: a `StereoShaderBase` buys nothing — the genuinely shared math (`stereo`, `stereo_noise_warp`, `pole_attenuation`, the `STEREO_PATTERN_ARG_LIMIT` clamp) already lives in core, while `project()`, the `sample()` pattern, and `Params::lerp` are deliberately different per effect. The "kept in sync by hand / propagate fixes across both" note was actively wrong (a Liquid2D pattern fix must NOT propagate to Flyby's grid). Rewrote both header notes to describe them as independent effects sharing core primitives, and softened the in-`sample()` "kept in sync" comment to scope it to the clamp guard. The warp time-scales (0.5/0.3) are already documented at each `warp()`'s `@param`.*
6. ✅ `effects/Voronoi.h:119` — **Per-pixel k=2 KD-tree query over 400 sites, no coherence reuse.** The heaviest shader in the effect set (already forced to SAMPLES=1) re-queries from scratch though the nearest pair is piecewise-constant over large cell interiors. → Seed each query from the previous pixel's result, or run the full query only on pixels whose coarse-grid corners disagree. *Resolved via coarse-grid coherence: the nearest-pair identity is classified once per 8 px grid corner (`kCoherenceBlock`); an interior pixel whose four surrounding corners agree on the same `{lo, hi}` pair skips the tree query and shades from two exact dot products. Output is bit-identical to the full query wherever the agreed pair is genuinely the nearest two (the shading — color blend + border test — stays per-pixel and exact; only the discrete query is elided). Seams and dense near-pole blocks have disagreeing corners and fall back to the full query, so the scheme self-corrects where cells shrink below a block. The render loop is open-coded (mirroring `Scan::Shader::draw<W,H,1>`'s clip iteration + `ScopedRenderTimer`) because the coarse grid needs integer pixel coordinates the generic shader callback does not expose. Residual approximation: a cell smaller than a block and nested between four agreeing corners can be dropped — a speck, acceptable atop the existing SAMPLES=1 point sampling.*
7. ✅ `effects/GSReactionDiffusion.h:70` / `effects/BZReactionDiffusion.h:73` — **RD reaction-term stability asserted, not bounded; BZ has no stability analysis.** GS bounds only the diffusion term and assumes the autocatalysis/feed-kill Jacobian is sub-dominant across a wide slider space; BZ ships no analysis and a 2× larger diffusion max. The Q8/Q16 post-step clamp is the actual blow-up backstop. → Document that the fixed-point clamp is the stability backstop (Euler bound is visual-quality only), or tighten BZ's diffusion slider to a documented bound.
8. `effects/ShapeShifter.h:51` — **`Count` (≤128) drives ~256 rasterizations/frame, unbudgeted.** `drawAll()` dispatches both a Plot and a Scan pass per layer; worst case is far beyond any device target (acknowledged in-code but shipped). → Clamp the effective loop count against a pixel budget, or gate the dual pass above a threshold.
9. `effects/GnomonicStars.h:106` — **Up to 2000 trig-heavy `fib_spiral` + SDF star rasterizations every frame, no cull.** Off-hemisphere or sub-pixel stars are still fully rasterized; the dominant per-frame cost at 288×144. → Horizon/size-cull after the Möbius warp before `Scan::Star::draw`.
10. ✅ `effects/HankinSolids.h:204` — **Per-frame persistent re-bind soundness rests on a debug-only assert.** `update_hankin` re-binds into `persistent_arena` every frame; correctness (counts never change, so `bind` reuses in place rather than growing the arena unboundedly) is enforced only by a contract assert that `NDEBUG` strips on the device. → Add an always-on `HS_CHECK` on first frame of each cycle that post-update vertex/face counts equal the compiled counts.
11. ✅ `effects/Thrusters.h:209` — **Unguarded normalize of a possibly-degenerate cross product.** Under large warp the two oriented vectors can become near-parallel, making `.normalized()` produce NaN/inf that seeds `Animation::Rotation`. → Guard the cross length and fall back to a fixed axis, mirroring `make_rotation`.
12. ✅ `core/filter.h:1339` — **`col_used[W]` is conditionally-live; one edit from an uninitialized-stack read.** It is filled only when `xc.active`, yet read at :1378 — currently safe only because short-circuit evaluation never reaches the read on the full-width path. → Default-initialize (`bool col_used[W] = {};`) or route the read through a self-guarding helper.
13. ✅ `driver.js:370` (daydream) — **Stale axis/effect labels linger when the live label count drops to zero.** `labelRenderer.render()` is skipped at `activeCount === 0`, but CSS2DRenderer only hides elements during a render pass, so previously-shown label `<div>`s stay visible on the N→0 transition. → Render once on the N→0 transition (track a `_hadLabels` flag).
14. ✅ `segment_worker.js:112` (daydream) — **Worker ignores `engine.setResolution()`'s false return, then clips to a size the engine isn't at.** Updates `canvasW/H`/`segRange` and re-clips unconditionally; on an unsupported size the engine stays put while the worker extracts for the new geometry. Not currently reachable (controller sends only valid presets). → Gate the update on the return value, mirroring `daydream.js`.
15. `tests/test_scan.h`, `tests/test_plot_scan.h` — **`Scan::Volume`/`TransformedVolume` and `Plot::SplineChain`/`ParticleSystem` have zero direct tests.** The volumetric ray-march (Raymarch) and spline/particle plotters are reached only via the smoke loop, which asserts no per-shape correctness. → Add direct tests pinning the ray-march silhouette/hit registers and the spline's control-point passage and tension deflection.

## 6. Overall Appraisal *(written independently of the prior review)*

**What this project actually is.** Holosphere is a single C++ rendering engine that drives three radically different targets from one source: a 600 MHz bare-metal microcontroller painting a sphere with a spinning LED strip at 480 RPM; a four-board distributed version of the same that keeps four independent CPUs phase-locked over a single wire with no shared clock; and a WebAssembly build that reproduces the identical imagery in a browser, pixel-for-pixel, as a design and preview tool. It carries 27 original generative effects spanning reaction-diffusion on a Fibonacci lattice, Hopf fibrations, spherical harmonics, Conway-operator polyhedral geometry with Islamic/Hankin star patterns, Möbius-warped feedback, and SDF ray-marching — all rendered in 16-bit linear light with OKLCH perceptual palette interpolation, on a device with ~335 KB of usable RAM and no heap to spare.

**The magnitude of the achievement is large, and it is large along an unusual axis.** The hard, rare thing here is not any one effect — it is the *engineering coherence* that lets all of this share one codebase without the abstractions either leaking or collapsing. Compile-time `<W,H>` parameterization gives the microcontroller zero-overhead specialization while the simulator instantiates a different resolution from the same templates. A partitioned arena with RAII scratch scopes replaces the heap entirely, with every allocation seam trapping and every budget documented to the byte. A fail-fast doctrine (`HS_CHECK`) is applied with genuine discipline about *where* — cold seams trap, hot loops don't — and is then *verified* by a death harness that re-execs the binary and asserts each trap fires with the exact illegal-instruction signal. That last move is the tell: most projects that talk about fail-fast never prove their traps work; this one tests its own safety net. The single most impressive subsystem, the Phantasm 1-wire sync, derives column position from the CPU cycle counter rather than counting interrupts, which makes masked-IRQ windows lossless *by construction* and the 32-bit counter wrap structurally unobservable — and it puts every load-bearing protocol decision in host-testable pure code under a thin ISR shell, then exercises it with a fault-injecting four-board simulator. That is research-grade systems engineering.

**Technical merit, relative to peers.** Against the hobbyist/maker POV-display and LED-art landscape, this is not in the same category — it is several tiers above. The typical artifact there is an Arduino sketch with `delay()`s, 8-bit sRGB blending, global state, and no tests; Holosphere blends in linear light, eliminates the heap, templates the whole pipeline, and ships a native test suite with sanitizers, a death harness, and CI smoke that drives every effect at both resolutions. Against *professional* embedded-graphics and creative-coding offerings, it holds up on the dimensions that matter most: the memory model, the determinism contract, the ISR/concurrency reasoning, and the documentation are at or above commercial firmware standards, and the sim/device bit-equivalence discipline (seeded `mt19937(1337)`, bit-faithful FastLED mocks, fixed-point round-trips pinned by golden vectors) is something most commercial teams aspire to and few enforce. The visible gap to a flat A is not architecture or craft — it is that test *coverage* trails test *infrastructure*: the newest and most numerically delicate paths (volumetric ray-march, feedback warp, RD stability, two-thirds of the effects) are validated by smoke and the human eye rather than by oracle.

**Academic and state-of-the-art context.** The individual algorithms are largely known art — OKLab/OKLCH is Ottosson's, the SDF/CSG and sphere-tracing vocabulary is Quílez's, the reaction-diffusion and Conway-operator and Hankin-pattern work all have literatures. The originality is not in inventing those; it is in the *integration* — running this entire menagerie in real time, deterministically, on a microcontroller, against a browser twin, with a one-wire distributed-clock protocol holding four boards together. Spherical POV displays exist as research demos and as products, but a unified, host-testable, browser-mirrored engine of this breadth and this code quality is not something I can point to a clear public peer for. The sync subsystem in particular would be a credible systems-paper artifact on its own.

**Artistic merit.** The effect set is genuinely strong and, importantly, *curated* rather than padded — the palette and color pipeline (perceptual interpolation, shortest-arc hue, gamut-preserving clipping, soft alpha in linear light) is doing real aesthetic work that cheaper pipelines can't, and it shows up exactly where the README claims (soft gradients, multi-layer compositing). The medium itself — full-color imagery floating on the surface of a physical spinning sphere — is striking, and the browser simulator lowers the barrier to iterating on it. This is the work of someone who is both a serious systems engineer and a serious visual artist, and the two halves reinforce rather than compromise each other.

**Bottom line.** Holosphere is an A− project whose distance to an A is almost entirely a coverage frontier, not a quality deficit. As an act of sustained, disciplined, single-author (or small-team) engineering across firmware, distributed real-time systems, graphics, color science, a WASM bridge, and a web client — held to a self-imposed standard that it then verifies rather than assumes — it is an unusually accomplished piece of work. It comfortably exceeds its hobbyist peers, stands with professional firmware on the axes that count, and in its sync subsystem and its sim/device-equivalence rigor reaches into territory that is genuinely state-of-the-art for the niche. The achievement is not a clever effect; it is that none of the seams show.
