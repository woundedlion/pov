# Holosphere — Comprehensive Code Quality Review

**Date:** 2026-06-21 (twelfth audit; supersedes the eleventh audit, whose findings 1–105 were reconciled and are nearly all marked resolved/rejected in the tree — see §5).
**Tree:** Holosphere `d6ce032`; daydream reviewed at its current `master` (`10c8d24`).
**Scope:** The Holosphere C++ engine and firmware — `core/` (rendering engine), `effects/` (27 effects + the shared `ReactionDiffusionBase.h`), `hardware/` (POV/DMA/1-wire-sync drivers), `targets/` (WASM bridge + Phantasm entry), `tests/` (native Clang suite), build/CI tooling (`CMakeLists.txt`, presets, `scripts/`) — **and the daydream web simulator** (`daydream.js`, `driver.js`, `state.js`, `gui.js`, `sidebar.js`, `recorder.js`, the segmented-POV worker pool, and the standalone geometry tools). Out of scope by instruction: `core/effects_legacy.h`, `targets/Holosphere/Holosphere.ino`, `core/rotate.h`. `core/FastNoiseLite.h` and `core/inplace_function.h` (vendored), and `core/color_luts.h` + `core/reaction_graph.cpp` (generated data) were assessed for integration/provenance only, not line-by-line.
**Method:** Nineteen independent component reviewers, each reading every in-scope file in full with the README as shared architectural context, grading objective quality (correctness, UB, ISR/memory-ordering, bounds, overflow, performance, testability) and subjective quality (architectural elegance, interface expressiveness, idiom). The prior `CODE_REVIEW.md` was deliberately **not** consulted during examination; it was reconciled only afterward (§5). The overall appraisal in §6 is written independently of the prior review.

## 1. Executive Summary

Read cold against the current tree by nineteen reviewers spanning both repositories, the Holosphere/daydream product holds the high bar its audit series has established. **No critical defect, and no correctness bug that ships a wrong frame on the default single-instance render path, was found.** Every reviewer of the C++ core independently reported the same striking pattern: the residual defects are overwhelmingly **internal inconsistencies with the codebase's own exacting standards** rather than ordinary bugs — the one free-function divisor lacking the `> 0` guard its templated sibling enforces, the one `.size()`-derived index taken before its `emplace_back` where every sibling uses `size()-1` after, the one quantize site that truncates where every other rounds, the one effect whose shape-morph timer skips the pause gate its preset timer honors. When a codebase's flaws are best described as exceptions to its own rules, the rules have largely won.

This pass's *new* information is concentrated in two places the eleventh audit had not surfaced:

1. **A reproducibility gap at the engine's one large generated table.** `core/reaction_graph.cpp` is a 92,160-byte Fibonacci-lattice K-NN table labeled "auto-generated — do not edit," but **no generator script exists in the repository** (unlike `color_luts.h`, which has `generate_luts.py` and a CI provenance gate). The runtime `node()`/`find_nearest_node()` must agree bit-for-bit with the offline generator that built the table, yet that agreement is unverifiable and unreproducible — and `node()` computes `y`/`radius` in `float` where the table's stated provenance is double-precision, a latent correctness question no current test can catch (the round-trip test seeds at the answer). This is the single most consequential finding of the pass and the only one rated High.

2. **The JS↔C++ "single source of truth" claim is partly unenforced.** The daydream geometry tools advertise (README §10.11) that splines and palettes run through the *same* engine math. For splines and solids that is true at runtime, but the spline tests inject *fake* evaluators and never execute the real exported `spline_cubic_*`; and `color.js`/`palette_math.js`/`lissajous_math.js` are **hand-ported copies** of the C++ color/HSV/lissajous math with no executed golden-vector cross-check. An engine-side change to the OKLab matrices or the HSV sextant path would not fail any JS test. The `segment_crosscheck` parity test is excellent — and it is the *only* executed parity check.

Everything else this pass found is Low-severity hardening, and the great majority of the eleventh audit's fix list is resolved in the tree.

**The grade is A−.** As in every prior audit, the distance to a flat A lives almost entirely at the **test-coverage frontier** — not the test *infrastructure*, which is A (forked death harness proving every `HS_CHECK` trap fires with the exact illegal-instruction signal; fault-injecting four-board sync simulator; injected-clock cross-run determinism; analytic oracles over Euler characteristic, KD-tree brute force, SDF cull-conservativeness). The gap is that the most numerically delicate paths remain unpinned (21 of 27 effects have only smoke + determinism; live SDF rasterization is end-to-end-tested only for `Scan::Ring`; the JS↔C++ parity surface above) — plus a thin band of latent, mostly-unreachable hazards.

**Verdict: A− overall.** The math/color/memory core, the mesh subsystem, the hardware protocol layer, the WASM bridge, the build/CI engineering, and the test *infrastructure* each grade A or above on their own terms; the daydream client matches the engine's discipline (the WASM detach-after-heap-growth hazard, the worker generation fence, and the fault-latch deadlock-breaker are all handled correctly and defensively).

## 3. Component Assessments

**core: math / geometry / constants / concepts / util** — Professionally-engineered numerical code and the strongest component on documentation (A+). Pole/antipode/singularity handling is systematic: `quaternion_from_basis` uses Shepperd's largest-pivot method (singularity-free), stereo/gnomonic pole sentinels carry matched forward/inverse thresholds, `fast_atan2` guards the origin, and the `normalized()`-trap / `normalized_or()`-fallback split is applied per call site with stated justification. The `FunctionRef`/`StoredFunctionRef`/`PipelineRef` type-erasure trio reasons explicitly about the rvalue-binding tradeoff and the code-size case for per-pixel indirection. No Critical/High; residuals are micro-nits (the hot projection path using the runtime `phi_to_y` rather than its templated overload; `fwd_distance` missing the precondition assert its siblings carry).

**core: spatial / mesh / static_circular_buffer** — A mature, self-consistent subsystem (A). Index-narrowing discipline is exemplary and centralized (`narrow_index`/`narrow_face_count` with the INT16/UINT16 distinction correctly drawn), the owned-vs-borrowed `MeshState` accessor split is consistently routed and move-clears the moved-from views, and `StaticCircularBuffer::construct_in_place` correctly `std::launder`s and commits indices only after a possibly-throwing ctor. KDTree build/search match a brute-force oracle. Residuals are all Low: a missing `static_assert(N > 0)`, three near-identical mesh-copy loops that risk drift, a coincident-vertex degenerate that traps in `classify` rather than degrading.

**core: conway / hankin / solids / generators** — Topology is sound across every Conway operator (dual/kis/ambo/truncate/expand/chamfer/snub), and the pool-sizing arithmetic was traced operator-by-operator and matches actual emission exactly. Every hand-rolled half-edge walk carries an always-on anti-hang `HS_CHECK`, so a corrupt `.next`/`.pair` chain traps on the bench instead of hanging the spinning device. Residuals are readability/maintainability: `relax` re-rolls the 1-ring orbit walk with a divergent step rule instead of reusing `vertex_orbit`; the Platonic/Archimedean registry slices use magic literals the size `static_assert` doesn't cross-check; transitive `using namespace` in `IslamicStarPatterns`.

**core: sdf / scan / plot (rasterization)** — The scanline-culling architecture is genuinely good: analytic per-row interval generation avoids `distance()` on empty columns, shared emitters keep byte-identical math in one place, and the seam-coalescing logic (negative-start re-wrap, the `static_assert`-proven 2× split bound, the integer-pixel `last_x2` clamp against double-alpha) is carefully reasoned. CSG interval logic correctly distinguishes "child fell back to full-scan" from "child covers row." Residuals are edge-case polish: a thin pole-adjacent `Ring` whose centerline clears the clamp guard but whose outer edge wraps the pole, and `Face` azimuth intervals lacking the polygon family's explicit one-pixel AA-fringe pad.

**core: canvas / filter / styles / led** — The variadic `Pipeline<W,H,Filters...>` with compile-time World↔Screen↔Pixel domain dispatch is the strongest single design in the engine; the fold-based `static_assert`s encode the non-obvious ordering invariants (World-before-Screen, terminal-last, unit-input-before-Mobius/Hole) directly into the type system. The double-buffer non-race proof is correct for the single-core/strict-index-ownership model, and the Feedback seam-recentering delta encoding is a subtle, correct fix. Residuals are Low: a `get_pixel` reads-previous-buffer footgun (documented), and positional preset aggregates in `styles.h` that a field reorder could silently shift.

**core: color / palettes / color_luts** — The linear/sRGB and OKLab/OKLCH stack is correct and thoughtfully factored: 16-bit linear blending, shortest-arc hue interpolation with achromatic-endpoint handling, a chroma-preserving gamut clip gated behind an in-gamut fast path, no `powf`/`atan2` on any hot path. The generated LUTs were verified to reproduce the reference transfer functions bit-for-bit including boundary entries; `lerp16`'s shift-reconstruction is exact at both endpoints and overflow-free; every float→int cast is NaN-clamped. Residuals are Low: `Color4::lerp` truncates its color weight where the rest of the file rounds; a few `ProceduralPalette` cosine amplitudes exceed 1.0 (clamped, so safe).

**core: animation / transformers / easing / waves** — Exceptional defensive commentary and compile-time-enforced borrow contracts (deleted rvalue overloads), correct `std::launder`/`iface`-upcast `TimelineEvent` type erasure, and an O(active) orientation-collapse pre-pass that is a genuinely subtle correctness property. The three wave generators are made interchangeable (all sit at the trough at t=0/phase=0, verified). The single code-level item is `RandomWalk::step` normalizing a cross product unguarded where its siblings guard; the rest are documentation-precision notes (decorrelation channels sharing a time base, `MobiusWarpEvolving` lacking `Noise`'s float-quantization note).

**core: memory / platform / inplace_function** — Best-in-class on clarity (A+) and the safety foundation of the whole engine. Arena overflow math is defense-in-depth (subtractive no-wrap bounds plus an independent multiply-overflow trap), alignment is validated before use, ScratchScope LIFO is structurally enforced, and the two-stamp debug lifetime tracking (arena generation for resets + per-vector rebind generation for grows) correctly distinguishes the two dangling modes. The `platform.h` FastLED mocks are bit-faithful, not approximations. Residuals are Low: `addmod8(_,_,0)` SIGFPEs on host where neighbors guard div-by-zero; the default `Fn` capacity `16` is duplicated across three sites that must stay in lockstep.

**core: engine / effect_registry / presets / reaction_graph** — Self-registration is textbook-correct: a Meyers singleton sidesteps the static-init-order fiasco, and `__attribute__((used, retain))` defends the dynamic initializer against both LTO elision and `--gc-sections`. The resolution dispatch turns an unhandled `<W,H>` into a compile error via the dependent-false idiom. This is also where the pass's one High lives: the generated K-NN table has no checked-in generator and `node()` mixes float/double against a double-precision table. Lesser items: `registerParam` doesn't validate `min <= max`; `Presets` exposes mutable indices publicly; adding a resolution requires three coupled edits.

**effects (27)** — They lean on engine primitives (KDTree, ParticleSystem, Plot/Scan, baked palettes, `MeshCarousel`, `deep_tween`, OrientationTrail) rather than reinventing them; the CRTP `ReactionDiffusionBase` is exemplary factoring. Numerical hardening is thorough and *documented why* (stereographic pole epsilons, empty-kernel divide guards, multi-day phase wrapping, NaN-cull-slip reasoning). `Dynamo`, `MindSplatter`, and `Voronoi` (with its new seam-stable coarse-grid coherence) are standouts. The weak dimension is performance budgeting on a few effects whose per-frame cost scales with a slider, plus a scattering of consistency nits (a radius-0 ring rasterized every frame in `Moire`; a shape-morph timer skipping the pause gate in `MeshFeedback`).

**hardware** — The crown jewel. The flywheel-from-time architecture (deriving column position from the CPU cycle counter rather than counting ISR fires) makes masked-IRQ windows lossless *by construction* and the 32-bit counter wrap structurally unobservable via per-half-rev epoch rebase. The single-writer ISR model is rigorously enforced; the `relaxed`-atomics-plus-cache-flush coherence split is the rare codebase that gets the "don't upgrade to acquire/release, it only adds DMB cost" nuance right; the count-coded odd-only symbol alphabet moves single-edge errors to "missed, never wrong"; pure host-testable cores are cleanly split from device shells. Residuals are all Low and latent: an `epoch_emits_left_` decrement-before-schedule that can erode redundancy during a coast, and `Config::valid()` not rejecting a negative `epoch_repeats`.

**targets: wasm / build / scripts** — Single-source-of-truth discipline is exemplary and pervasive (param order, resolution list, effect roster all derive from one expansion). The zero-copy memory-view contract is understood end-to-end (pre-sized once, never realloc'd, re-fetched on detach; the param vectors are `reserve`d and only `clear`ed so no boundary call can trigger detaching heap growth), `MAX_PARAMS` and every resolution are pinned by `static_assert`, and the untrusted-JS boundary consistently rejects-and-returns. Gaps are in CI breadth (the MeshOps `MESHOP_1F`/`relax`/`hankin` embind operators ride uncovered) and a Windows CRLF wrinkle in the LUT generator's stdout path.

**daydream: core** — Matches the engine's discipline. The hardest hazard in a zero-copy WASM bridge — detach-after-heap-growth — is handled rigorously and in the right places (`refreshPixelView` detects `byteLength === 0` and re-points all aliases; `instanceColor.array` is nulled before `dispose()`), the three-way alias invariant is asserted rather than assumed, URLSync is injection-safe and race-free, and listener/observer/worker/GPU teardown is genuinely symmetric. Residuals are Low: the init-path `setResolution` return ignored, a `getParameters()` comment referencing a nonexistent method, an un-`catch`ed clipboard write.

**daydream: UI & recording** — The session-local closure binding in `recorder.start()` is a genuinely correct solution to the stop→start race that MediaRecorder's async `onstop` creates, and `selectMimeType` reads back the actual `mimeType` so codec fallback never mislabels the file. Sidebar roving-tabindex is correct and accessible. The one Medium is a native-resolution recording that has no guard against a mid-recording canvas resize (the offscreen path handles it; native doesn't); the rest are Low (unvalidated `addColor` hydration, silently-latched mid-recording setters).

**daydream: segmented workers** — The generation fence (snapshot-at-dispatch, compare-at-receipt, drop-before-composite) is a clean, correct solution to stale-frame compositing, and the fault latch that breaks the would-be deadlock when a WASM trap fires `onerror` but never `frame` is well-reasoned and fully tested. Zero-copy transfer discipline is correct (fresh per-frame buffers, no cross-frame aliasing), the worker message queue serializes a `setResolution` arriving mid-`init`, and the protocol typedef is in sync with both endpoints. Residuals are Low/latent: an in-flight render promise never settled on `destroy()`.

**daydream: tools & JS tests** — The tool math matches the engine where it must (lissajous identical to `geometry.h`, `hsvToRgb` ports the integer `>>8` sextant path byte-for-byte, the OKLab matrices match Ottosson), and `segment_crosscheck.test.js` is a rigorous parity check against a faithful re-port of `pov_segment_map.h`. The substantive gap is that `segment_crosscheck` is the *only* executed parity test: the spline tool tests inject fake evaluators and the color/palette/lissajous modules are hand-ported with no golden-vector cross-check, so the README §10.11 "single source of truth" claim is unenforced for everything except segment layout.

### High

1. ✅ `core/reaction_graph.cpp` (whole table) + `core/reaction_graph.h:38` — **The generated Fibonacci-lattice K-NN table has no checked-in generator, and the runtime computes `node()` in mixed float/double against a double-precision table.** The 92,160-byte `neighbors[7680][6]` is labeled "auto-generated — do not edit," but a repo-wide search finds no script that emits it (`generate_luts.py` produces only `color_luts.h`). The runtime `node()`/`find_nearest_node()` must agree bit-for-bit with the offline generator — the header comment justifies folding `theta` in double precisely "to agree with the offline double-precision generator that built neighbors[]" — yet `y` and `radius = sqrtf(1 - y*y)` stay `float`, so near the poles the runtime can disagree with the table by more than a neighbor spacing and `find_nearest_node` could converge to a different owner. With no generator and no regenerate-and-diff test, the agreement is unverifiable, unreproducible, and silently stale if anyone touches `golden_angle`/`RD_N`/the `node()` formula; the round-trip test cannot catch it because it seeds at the answer. → Check in `scripts/generate_reaction_graph.py` that emits `reaction_graph.cpp` from the *same* `golden_angle`/`node()` math, add a CI regenerate-and-diff gate mirroring the `color_luts` provenance job, and compute `y`/`radius` in double and narrow once (symmetric with `theta`) — or document that the generator used float for `y` so the two provably match.
   **✅ Resolved.** Recovered `scripts/generate_reaction_graph.py` (pure-Python, dependency-free like `generate_luts.py`); it reproduces the committed table **bit-for-bit in double precision** (7680/7680 rows, including neighbor sort order). float32 `y`/`radius` reproduces the same neighbor *sets* but flips the sort order of 5 near-tie rows — so the provenance is genuinely double, and the correct fix is option A, not option B (the generator did **not** use float). `node()` now folds `y`/`radius`/`theta` in double and narrows once into the float `Vector`, symmetric with the generator. Added the `reaction-graph-provenance` CI job (numeric-token diff, mirroring `lut-provenance`) and a self-guarding latitude-window assertion in the generator so a future `RD_N` bump aborts rather than emitting a biased table.

### Medium

2. `tools/spline_math.js` + `tests/spline_math.test.js` — **The JS↔C++ spline parity (README §10.11's "single source of truth") is never executed.** The Catmull-Rom/Bézier tests inject fake evaluators (`lerpEval`, `passThroughTangents`); the real exported `spline_cubic_fast`/`spline_cubic_slerp`/`spline_catmull_rom_tangents` are never invoked, so only loop bookkeeping is tested and the highest-drift-risk boundary is unverified. → Add a WASM-backed test (à la `wasm_smoke.mjs`) that samples a known control-point set through the real exported evaluators and pins the output, or crosschecks against a C++ reference vector.

## 6. Overall Appraisal *(written independently of the prior review)*

**What this project actually is.** Holosphere is one C++ rendering engine that drives three radically different targets from a single source of truth: a 600 MHz bare-metal Teensy 4.x painting a sphere with a strip of LEDs spinning at 480 RPM; a four-board distributed sibling (Phantasm) that keeps four independent CPUs phase-locked over a single wire with no shared clock; and a WebAssembly build that reproduces the identical imagery in a browser as a design tool. On top of that substrate sit 27 original generative effects — reaction-diffusion on a Fibonacci lattice, Hopf fibrations, spherical harmonics, Conway-operator polyhedra dressed in Hankin/Islamic star patterns, Möbius-warped feedback, SDF ray-marching — all rendered in 16-bit linear light with OKLCH perceptual palette interpolation, on a device with ~335 KB of usable RAM and no heap to spend.

**The magnitude of the achievement is large, and it is large along an unusual axis.** The rare, hard thing here is not any single effect; it is the *engineering coherence* that lets a firmware target, a distributed real-time system, and a browser application share one codebase without the abstractions either leaking or collapsing. Nineteen independent reviewers — most of whom saw only their slice of the tree — converged on the same verdict by the same route: the way you find a flaw in this code is to locate the one place a self-imposed rule *isn't* honored. That is the signature of a codebase whose discipline is real rather than aspirational. Compile-time `<W,H>` parameterization gives the microcontroller zero-overhead specialization while the simulator instantiates a different resolution from the same templates. A partitioned arena with RAII scratch scopes replaces the heap entirely, every allocation seam trapping and every budget documented to the byte. A fail-fast doctrine (`HS_CHECK`) is applied with genuine discipline about *where* — cold seams trap, the per-pixel hot loop never does — and then *verified* by a death harness that re-execs the binary and asserts each trap fires with the exact illegal-instruction signal. That last move is the tell: most projects that talk about fail-fast never prove their traps work; this one tests its own safety net, and proves the net is load-bearing rather than decorative.

**Technical merit, relative to peers.** Against the hobbyist/maker POV and LED-art landscape this is not in the same category — it is several tiers above. The typical artifact there is an Arduino sketch with `delay()`s, 8-bit sRGB blending, global mutable state, and no tests; Holosphere blends in linear light, eliminates the heap, templates the whole pipeline, mocks FastLED bit-faithfully so the host build is the device build's twin, and ships a sanitizer-instrumented native suite with a death harness and CI that drives every effect at both resolutions. Against *professional* embedded-graphics and creative-coding offerings it holds up on the dimensions that matter most — the memory model, the determinism contract (seeded `mt19937(1337)`, fixed-point round-trips pinned by golden vectors), the ISR/concurrency reasoning, and the documentation are at or above commercial-firmware standards, and the `relaxed`-atomics-plus-cache-flush memory-ordering analysis is more careful than most shipping firmware bothers to be. The single subsystem I would hold up without qualification is the Phantasm 1-wire sync: deriving column position from the CPU cycle counter rather than counting interrupts makes masked-IRQ windows lossless *by construction* and the 32-bit counter wrap structurally unobservable, with every load-bearing protocol decision placed in host-testable pure code under a thin ISR shell and then exercised by a fault-injecting four-board simulator. That is research-grade systems engineering, and it would be a credible systems-paper artifact on its own.

**Academic and state-of-the-art context.** The individual algorithms are largely known art — OKLab/OKLCH is Ottosson's, the SDF/CSG and sphere-tracing vocabulary is Quílez's, and reaction-diffusion, Conway operators, and Hankin patterns each have literatures. The originality is not in inventing those; it is in the *integration* — running this entire menagerie in real time, deterministically, on a microcontroller, against a pixel-faithful browser twin, with a one-wire distributed-clock protocol holding four boards together and a death-harness-verified safety contract underneath. Spherical POV displays exist as research demos and as products, but a unified, host-testable, browser-mirrored engine of this breadth and this code quality is not something I can point to a clear public peer for.

**Artistic merit.** The effect set is genuinely strong and, importantly, *curated* rather than padded — the color pipeline (perceptual interpolation, shortest-arc hue, gamut-preserving clipping, soft alpha in linear light) is doing real aesthetic work that cheaper pipelines cannot, and it shows up exactly where the README claims (soft gradients, multi-layer compositing). The medium itself — full-color imagery floating on the surface of a physical spinning sphere — is striking, and the browser simulator collapses the iteration loop for an artifact that is otherwise expensive to see. This is the work of someone who is simultaneously a serious systems engineer and a serious visual artist, and the two halves reinforce rather than compromise each other.

**Bottom line.** Holosphere is an A− project whose distance to an A is almost entirely a coverage frontier, not a quality deficit. The one finding this pass would actually act on before the next show is mundane and fixable: check in the reaction-graph generator and a regenerate-and-diff gate, so the engine's one large generated table is reproducible rather than a one-way artifact. Everything else is hardening at the margins of a codebase that has already won its own argument. As an act of sustained, disciplined engineering across firmware, distributed real-time systems, graphics, color science, a WASM bridge, and a web client — held to a self-imposed standard that it then verifies rather than assumes — it is an unusually accomplished piece of work. It comfortably exceeds its hobbyist peers, stands with professional firmware on the axes that count, and in its sync subsystem and its sim/device-equivalence rigor reaches genuinely state-of-the-art territory for the niche. The achievement is not a clever effect; it is that, across three targets and two languages, none of the seams show.
